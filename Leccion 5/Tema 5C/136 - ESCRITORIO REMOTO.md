
Una VPN de acceso remoto une el equipo o teléfono inteligente del usuario a una red privada remota a través de un túnel seguro mediante una red pública. El acceso remoto también puede ser un medio para conectarse a un equipo específico a través de una red. Este tipo de acceso remoto implica conectarse a un servidor de terminal en un host mediante un software que solo transfiere datos de la terminal. La conexión podría ser entre un cliente y un servidor de terminal en la misma red local o entre redes remotas.

Una herramienta gráfica de acceso remoto envía datos de pantalla y audio desde el host remoto al cliente y transfiere la entrada del mouse y del teclado del cliente al host remoto. El protocolo de escritorio remoto (RDP) de Microsoft puede utilizarse para acceder a una máquina física de forma individual. 

Como alternativa, un sitio puede operar una puerta de enlace de escritorio remoto que facilita el acceso a escritorios virtuales o aplicaciones individuales que se ejecutan en los servidores de la red. Las conexiones RDP están cifradas de forma predeterminada. Existen varias alternativas populares a los escritorios remotos. La mayoría de las soluciones admiten el acceso remoto a plataformas que no son Windows, como macOS, iOS, Linux, Chrome OS y Android, por ejemplo. Algunos ejemplos son TeamViewer (teamviewer.com/en) y Virtual Network Computing (VNC), que es implementado por varios proveedores diferentes (notablemente realvnc.com/en).

En el pasado, estos productos de escritorio remoto requerían una aplicación de cliente dedicada. Ahora, el acceso remoto al escritorio puede realizarse mediante un cliente de navegador web. El elemento canvas (lienzo) introducido en HTML5 permite que un navegador dibuje y actualice un escritorio con relativamente poco retraso. También puede manejar audio. Esto se conoce como HTML5 VPN o puerta de enlace de escritorio remoto sin cliente (guacamole.apache.org). Esta solución utiliza un protocolo llamado WebSockets, que permite enviar mensajes bidireccionales entre el servidor y el cliente sin requerir la sobrecarga de peticiones HTTP separadas.
