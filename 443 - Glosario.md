
Glosario

Política de uso aceptable (AUP): una política que regula el uso por parte de los empleados de los equipos de la empresa y los servicios de Internet. Los proveedores de servicios de Internet (ISP) también pueden aplicar políticas AUP a sus clientes.

Tarjeta de acceso: un mecanismo de autenticación que permite a un usuario presentar una tarjeta inteligente para operar un sistema de entrada.

Lista de control de acceso (ACL): colección de entradas de control de acceso (ACE) que determina a qué sujetos (cuentas de usuario, direcciones IP de host, etc.) se les permite o se les niega el acceso para acceder al objeto y cuáles son los privilegios otorgados (solo lectura, lectura/escritura, etc.).

Vestíbulo de control de acceso: sistema de entrada seguro con dos puertas de acceso, de las cuales solo puede abrirse una puerta a la vez.

Punto de acceso (AP): dispositivo que facilita una conexión entre dispositivos inalámbricos y puede conectarse al cableado de red, implementando una red WLAN en modo infraestructura.

Bloqueo de cuenta: política que impide el acceso a una cuenta circunstancias específicas, como cuando se producen numerosos intentos fallidos de autenticación.

Políticas de cuenta: conjunto de normas que regulan la información de seguridad del usuario, como la caducidad y unicidad de las contraseñas, que se pueden aplicar a nivel global.

Registro: seguimiento del uso autorizado de los recursos o los derecho de uso por parte de un sujeto, y emisión de alertas cuando se detecta o se intenta un uso no autorizado.

Adquisición/compra de activos: conjunto de políticas y procesos diseñados para garantizar que las compras de activos así como los contratos de servicios se gestionen de manera integral y segura, que se utilicen proveedores y vendedores autorizados, y que cumplan con los objetivos comerciales.

Reconocimiento activo: técnicas de pruebas de penetración que interactúan directamente con los sistemas objetivo.

Control de seguridad activo: controles de seguridad de detección y prevención que utilizan un agente o una configuración de red para supervisar los hosts. Esto permite realizar escaneos de credenciales más precisos, aunque consume algunos recursos del host y pueden detectarlo los actores de amenaza.

Red ad hoc: tipo de red inalámbrica en la que los dispositivos conectados se comunican directamente entre sí en lugar de hacerlo a través de un medio establecido.

Protocolo de resolución de direcciones (ARP): mecanismo de difusión por el cual la dirección MAC de hardware de una interfaz se corresponde con una dirección IP en un segmento de red local.

Amenaza persistente avanzada (APT): capacidad de un atacante para obtener, mantener y diversificar el acceso a los sistemas de red mediante exploits y malware.

Adware: software que registra información sobre una PC y su usuario. El concepto de adware se utiliza para describir un software al que el usuario permite registrar información sobre sus hábitos.

Protocolo de modo de contador de Galois AES (GCMP): es un modo de operación de alto rendimiento para el cifrado simétrico. Proporciona una característica especial llamada cifrado autenticado con datos asociados o AEAD.

Air-gapped: tipo de aislamiento de red que separa físicamente un host de otros hosts o una red de todas las demás redes.

Ajuste de alertas: proceso modificación de reglas de detección y correlación para disminuir la incidencia de falsos positivos y alertas de baja prioridad.

Algoritmo: operaciones que transforman un texto plano en un texto cifrado con propiedades criptográficas, también denominado cifrado. Existen tipos de cifrado simétrico, asimétrico y hash.

Lista de permitidos: configuración de seguridad en la que se restringe el acceso a cualquier entidad (proceso de software, IP/dominio, etc.) a menos que esté explícitamente incluida en una lista de permitidos.

Ataque amplificado: tipo de ataque basado en la red en el que el atacante incrementa significativamente el ancho de banda enviado a una víctima durante un ataque DDoS mediante el uso de un factor de amplificación.

Análisis: proceso de respuesta ante incidentes en el que se evalúan indicadores para determinar la validez, el impacto y la categoría.

Expectativa de pérdida anualizada (ALE): costo total de un riesgo para una organización en un período de un año. Esto se determina multiplicando la expectativa de pérdida individual (SLE) por la tasa anual de ocurrencia (ARO).

Tasa anualizada de ocurrencia (ARO): en el cálculo de riesgos, una expresión de la probabilidad o posibilidad de un riesgo como el número de veces por año que se espera que ocurra una pérdida en particular.

Reconocimiento de comportamiento anómalo: detectan de forma automática usuarios, hosts y servicios que se desvían del comportamiento esperado, o sistemas y capacitaciones que promueven la denuncia de tales comportamientos por parte de los empleados.

Antivirus: inspeccionan el tráfico para localizar y bloquear los virus.

Escaneo de antivirus (A-V): software capaz de detectar y eliminar infecciones de virus y (en la mayoría de los casos) otros tipos de malware, como gusanos, troyanos, rootkits, adware, spyware, descifradores de contraseñas, asignadores de red, herramientas DoS, etc.

Cualquier cosa como servicio (XaaS): concepto de que la mayoría de los tipos de requisitos de TI se pueden implementar como un modelo de servicio en la nube.

Dispositivo cortafuegos: dispositivo de hardware independiente que realiza únicamente la función de un cortafuegos, que está integrado en el firmware del dispositivo.

Interfaz de programación de aplicaciones (API): métodos expuestos por un script o programa que permiten que otros script o programas lo utilicen. Por ejemplo, una API permite a los desarrolladores de software acceder a funciones de la pila de red TCP/IP bajo un sistema operativo particular.

Virtualización de aplicaciones: modelo de entrega de software en el que el código se ejecuta en un servidor y se transmite de manera continua a un cliente.

Ejecución de código arbitrario: vulnerabilidad que permite a un atacante ejecutar su propio código o un módulo que aprovecha dicha vulnerabilidad.

Envenenamiento de ARP: ataque basado en la red en el que un atacante con acceso al segmento de red local objetivo redirige una dirección IP a la dirección MAC de un equipo que no es el destinatario previsto. Esto se puede utilizar para realizar diversos ataques, como DoS, suplantación de identidad y en ruta (anteriormente conocido como ataque de hombre en el medio [MitM]).

Inteligencia artificial: la ciencia de crear máquinas con la capacidad de desarrollar estrategias de resolución y análisis de problemas sin una dirección o intervención humana significativa.

Activo: recurso que posee valor económico. Para efectos contables, los activos se clasifican de diferentes maneras, como tangibles e intangibles o a corto y largo plazo. La administración de activos consiste en identificar cada activo y registrar su ubicación, atributos y valor en una base de datos.

Algoritmo asimétrico: cifrado que utiliza claves públicas y privadas. Las claves están vinculadas matemáticamente con algoritmos de criptografía de curva elíptica (ECC) o Rivest, Shamir, Adleman (RSA), sin embargo, la clave privada no puede ser obtenida a partir de la clave pública. Una clave asimétrica no puede revertir la operación que realiza, por lo que la clave pública no puede descifrar lo que cifró, por ejemplo.

Superficie de ataque: puntos en los que una red o aplicación reciben conexiones externas o entradas/salidas que son vectores potenciales para que un actor de amenaza los vulnere.

Vector de ataque: ruta específica por la que un actor de amenazas obtiene acceso no autorizado a un sistema.

Atestación: capacidad de un autenticador u otro módulo criptográfico para demostrar que es una raíz de confianza y puede proporcionar informes confiables para demostrar que un dispositivo o equipo es una plataforma confiable.

control de acceso basado en atributos: técnica de control de acceso que evalúa un conjunto de atributos que posee cada sujeto para determinar si se debe otorgar acceso.

Autenticación: método para validar las credenciales únicas de una entidad o individuo en particular.

Autenticación, autorización y registro (AAA): concepto de seguridad en el que una plataforma centralizada verifica la identificación del usuario, garantiza que se le asignen los permisos correspondientes y luego registra estas acciones para crear una traza de auditoría.

Encabezado de autenticación (AH): protocolo IPSec que proporciona autenticación del origen de los datos transmitidos, así como integridad y protección contra ataques de repetición.

Autenticador: switch o enrutador PNAC que activa EAPoL y transmite los datos de autenticación de un suplicante a un servidor de autenticación, como un servidor RADIUS.

Autorizado: hacker que realiza pruebas de penetración autorizadas u otras consultorías de seguridad.

Autorización: proceso para determinar qué permisos y privilegios tiene una entidad específica.

Disponibilidad: el objetivo de seguridad fundamental que garantiza que los sistemas informáticos operen de manera continua y que las personas autorizadas puedan acceder a los datos que necesitan.

Puerta trasera: mecanismo para obtener acceso a un equipo que elude o subvierte el método normal de autenticación.

Respaldo: copia de seguridad de los datos de producción almacenada en medios extraíbles, por lo general con una cadencia regular. Los diferentes tipos respaldos (completo, progresivo o diferencial) equilibran la capacidad de los medios, el tiempo necesario para realizar el respaldo y el tiempo requerido para restaurarlo.

Generador respaldo: una fuente de energía de reserva que se alimenta por diésel o propano. En caso de que ocurra un corte de energía, un SAI debe proporcionar energía transitoria, ya que un generador de respaldo no se puede activar lo suficientemente rápido.

Configuración de línea de base: conjunto de configuraciones de ajustes y seguridad que se aplican a un sistema o red específica dentro de la organización.

Detección basada en el comportamiento: sistema de monitoreo de red que detecta cambios en las secuencias de datos de funcionamiento normal e identifica secuencias anormales.

Autenticación biométrica: mecanismo de autenticación que permite a un usuario realizar un escaneo biométrico para operar un sistema de entrada o acceso. Las características físicas almacenadas como una plantilla de datos digitales se pueden utilizar para autenticar a un usuario. Las características típicas utilizadas son: patrón facial, iris, retina, patrón de huellas dactilares y reconocimiento de firma.

Ataque de cumpleaños: tipo de ataque de contraseña que explota las debilidades en los algoritmos matemáticos utilizados para cifrar contraseñas, con el fin de aprovechar la probabilidad de que diferentes entradas de contraseñas produzcan la misma salida cifrada.

Chantaje: exigencia de dinero para evitar la divulgación de información.

Lista de bloqueo: configuración de seguridad en la que generalmente se permite el acceso a un proceso de software, IP/dominio u otra entidad, a menos que se indique como explícitamente prohibido en la lista.

Cadena de bloqueo: concepto en el que una lista en expansión de registros transaccionales enumerados en un libro contable público se asegura mediante criptografía.

Contenido bloqueado: un indicador potencial de actividad maliciosa donde los registros de auditoría muestran intentos no autorizados de lectura o copia de un archivo u otros datos.

Bluejacking: envío de mensajes o un imágenes no solicitados mediante una conexión Bluetooth.

Ataque: ataque inalámbrico en el que un atacante obtiene acceso a información no autorizada de un dispositivo mediante una conexión Bluetooth.

Pilotillos: postes verticales resistentes instalados para controlar el tráfico rodado o diseñados para evitar atracos con alunizajes y embestidas de vehículos.

Botnet (red de robots): grupo de hosts o dispositivos que ha sido infectado por un programa de control llamado bot, que permite a los atacantes explotar los hosts para lanzar ataques.

Traiga su propio dispositivo (BYOD): marco de seguridad y herramientas para facilitar el uso de dispositivos de propiedad personal para acceder a redes y datos corporativos.

Ataque por fuerza bruta: tipo de ataque de contraseña en el que un atacante utiliza una aplicación para probar de forma exhaustiva todas las combinaciones alfanuméricas posibles para descifrar las contraseñas cifradas.

Desbordamiento de búfer: ataque en el que los datos sobrepasan el límite del búfer de destino y comienzan a corromper la memoria adyacente. Esto puede permitir que el atacante bloquee el sistema o ejecute código arbitrario.

Bug bounty (programa de recompensas por detección de errores): esquema de recompensas operado por proveedores de software y servicios web para reportar vulnerabilidades.

Continuidad del negocio (BC): conjunto de procesos que permiten a una organización mantener el funcionamiento normal del negocio ante algún acontecimiento adverso.

Compromiso de correo electrónico empresarial (BEC): ataque de suplantación de identidad en el que el atacante obtiene el control de la cuenta de un empleado y lo utiliza para convencer a otros empleados de que realicen acciones fraudulentas.

Análisis de impacto del negocio (BIA): actividad sistemática que identifica los riesgos de la organización y determina su efecto en las operaciones en curso, de misión crítica.

Acuerdo de asociación comercial (BPA): acuerdo entre dos empresas para trabajar en estrecha colaboración, como los acuerdos de colaboración que establecen las grandes empresas de tecnología con revendedores y proveedores de soluciones.

Cables de seguridad: precintos y candados que se utilizan para asegurar físicamente los dispositivos contra el robos. Algunos sistemas también cuentan con placas frontales con traba, que evitan el acceso al interruptor de energía y unidades extraíbles.

Motor de almacenamiento en caché: una funcionalidad de muchos servidores proxy que les permite conservar una copia de las páginas web solicitadas con frecuencia.

Lista de llamadas: documento que enumera los contactos autorizados para recibir notificaciones y colaborar durante un incidente de seguridad.

Ataque de canonicalización: modalidad de ataque donde los caracteres de entrada se codifican de tal manera que evaden las medidas de validación de entrada vulnerables.

Planificación de capacidad: práctica que implica estimar los recursos humanos, de almacenamiento, de hardware, de software y de infraestructura de conexión necesarios durante un período futuro determinado.

Clonación de tarjetas: creación de una copia de una tarjeta de acceso sin contacto.

Celular: los estándares para implementar el acceso a los datos a través de redes celulares se desarrollan como generaciones sucesivas. Para 2G (hasta aproximadamente 48 Kb/s) y 3G (hasta aproximadamente 42 Mb/s), existen redes de proveedores GSM y CDMA de la competencia. Los estándares para 4G (hasta aproximadamente 90 Mb/s) y 5G (hasta aproximadamente 300 Mb/s) se desarrollan bajo estándares LTE convergentes.

Arquitectura de computación centralizada: modelo en el que todo el procesamiento y almacenamiento de datos se realiza en un solo lugar.

Cadena de certificados: método de validación de un certificado que consiste en rastrear cada autoridad de certificación (CA) que firma el certificado, siguiendo la jerarquía hasta llegar a la CA raíz. También se conoce como cadena de confianza.

Lista de revocación de certificados (CRL): una lista de certificados que fueron revocados antes de su fecha de vencimiento.

Solicitud de firma de certificado (CSR): archivo ASCII Base64 que un sujeto envía a una CA para obtener un certificado.

Certificación: técnica de eliminación de activos que confía en un tercero para utilizar métodos de saneamiento o destrucción para eliminar completamente los restos de datos, y proporcionar evidencia documental de que el proceso está completo y fue exitoso.

Cadena de custodia: registro del manejo de la evidencia, desde su recolección y presentación ante los tribunales hasta su eliminación.

Control de cambios: proceso mediante el cual se registra y aprueba la necesidad de un cambio.

Administración de cambios: proceso a través del cual se implementan los cambios en la configuración de los sistemas de información como parte de los esfuerzos generales de la gestión de configuración de la organización.

Director de Tecnologías de la Información (CIO): ejecutivo de la empresa, cuya responsabilidad principal es la gestión de activos y de procedimientos de tecnología de la información.

Director de Seguridad (CSO): por lo general, es el cargo de la persona responsable de garantizar la seguridad de la información y los sistemas de toda la empresa. También se conoce como director de Seguridad de la Información (CISO).

Director de tecnología (CTO): director ejecutivo de la empresa, cuya función principal es hacer un uso eficaz de las plataformas e innovaciones informáticas nuevas y emergentes.

chmod command: comando Linux para administrar permisos de archivos.

Elija su propio dispositivo (CYOD): un modelo de aprovisionamiento de dispositivos móviles empresariales en el que se les ofrece a los empleados una selección de dispositivos corporativos para el trabajo y, opcionalmente, para uso personal.

Tríada CIA: tres principios de control y administración de la seguridad. También se conoce como triada de seguridad de la información. También se conoce en orden inverso como la tríada AIC.

Suite de cifrado: listas de algoritmos criptográficos que un servidor y un cliente pueden utilizar para negociar una conexión segura

Texto cifrado: datos que se cifraron y que no se pueden leer sin la clave de cifrado.

Política de escritorio limpio: política de una organización que exige que las áreas de trabajo de los empleados estén libres de información potencialmente confidencial. Los documentos confidenciales no deben dejarse donde personal no autorizado pueda verlos.

Clonación: proceso de duplicar con rapidez la configuración de una máquina virtual cuando se necesitan varias máquinas idénticas de inmediato.

Privado/propietario: código de software o investigación de seguridad que permanece en la propiedad del desarrollador y solo se puede usar bajo condiciones de licencias permitidas.

Computación en la nube: arquitectura informática donde los recursos bajo demanda provistos con atributos de alta disponibilidad, escalabilidad y elasticidad se facturan a clientes sobre la base de utilización medida.

Modelo de implementación de la nube: clasificación que determina la propiedad y la gestión de una infraestructura de nube como pública, privada, comunitaria o híbrida.

Modelo de servicio en la nube: clasificación que determina el aprovisionamiento de servicios en la nube y el límite de la responsabilidad del proveedor de servicio en la nube como software, plataforma, infraestructura, entre otros.

Proveedor de servicios en la nube (CSP): organización que brinda servicios de infraestructura, aplicaciones o almacenamiento a través de una oferta centrada en la nube y basada en el modelo de suscripción “como servicio”.

Clustering (agrupación en clústeres): técnica de balanceo de carga en la que un grupo de servidores se configuran como una unidad y trabajan juntos para proporcionar servicios de red.

Código de conducta: el comportamiento profesional depende de estándares éticos básicos, como la honestidad y la equidad. Algunas profesiones pueden haber desarrollado códigos de ética para abordar situaciones difíciles. También pueden tener un código de ética para comunicar los valores esperados de los empleados.

Código de firma: método de utilizar una firma digital para garantizar la fuente y la integridad del código de programación.

Cold site (sitio frío): lugar alternativo predeterminado donde se puede reconstruir una red luego de una catástrofe.

Colisión: en criptografía, el acto de dos entradas de texto plano diferentes que producen exactamente la misma salida de texto cifrado.

Comando y control (C2): infraestructura de hosts y servicios con la que los atacantes dirigen, distribuyen y controlan los programas maliciosos a través de redes de robots.

Inyección de comandos: cuando un actor de amenaza es capaz de ejecutar comandos de shell arbitrarios en un host a través de una aplicación web vulnerable.

Nombre común (CN): atributo X500 que expresa un host o nombre de usuario. También se utiliza como identificador de asunto para un certificado digital.

Vulnerabilidades y exposiciones comunes (CVE): esquema de identificación de vulnerabilidades desarrollado por el MITRE y adoptado por el NIST.

Sistema común de puntuación de vulnerabilidades (CVSS): enfoque de gestión de riesgos para cuantificar los datos de vulnerabilidad y, a continuación, tener en cuenta el grado de riesgo de los distintos tipos de sistemas o información.

Nube comunitaria: plataforma en la nube que se implementa para uso compartido por suscriptores colaborativos.

Control compensatorio: medida de seguridad que asume la mitigación del riesgo cuando un control primario falla o no puede cumplir completamente con las expectativas.

Computación: recursos de procesamiento, memoria, almacenamiento y red que permiten que un host o dispositivo de red maneje una carga de trabajo determinada.

Equipo de respuesta a incidentes informáticos (CIRT): equipo responsable de la respuesta a incidentes. El CIRT debe tener experiencia en varios dominios comerciales (por ejemplo, TI, RR. HH., asuntos legales y marketing).

Programas de capacitación y educación en informática (TCC): capacitaciones impartidas mediante el uso de dispositivos informáticos, y modelos y diseño de instrucción de aprendizaje electrónico.

Uso de sesión concurrente: un indicador potencial de actividad maliciosa en la que una cuenta inicia múltiples sesiones en uno o más hosts.

Confidencialidad: el objetivo fundamental de seguridad es el de mantener la información y las comunicaciones privadas y protegerlas contra el acceso no autorizado.

Línea base de configuración: ajustes para la configuración de servicios y políticas para un dispositivo de red o un servidor que opera en un rol de aplicación específico (como servidor web, servidor de correo, servidor de archivo/impresión, etc.).

Administración de configuración: proceso mediante el cual los componentes de los sistemas de información de una organización se mantienen en un estado controlado que cumple con los requisitos de la organización, incluyendo los de seguridad y cumplimiento.

Conflicto de intereses: cuando un individuo u organización tiene inversiones u obligaciones que podrían comprometer su capacidad para actuar de manera objetiva, imparcial o en el mejor interés de otra parte.

Contenedorización: implementación de virtualización de un sistema operativo que contiene todo lo necesario para ejecutar un servicio, aplicación o microservicio.

Contención: proceso de respuesta a incidentes en el que el alcance de los sistemas afectados se restringe mediante técnicas y herramientas de aislamiento, segmentación y cuarentena.

Plan de continuidad de operaciones (COOP): identifica cómo los procesos comerciales deben enfrentar tanto las interrupciones menores como las de nivel de desastre al garantizar que exista redundancia de procesamiento que respalde el flujo de trabajo.

Plano de control: en la arquitectura de confianza cero, las funciones que definen la política y determinan las decisiones de acceso.

cookie: archivo de texto que se utiliza para almacenar información sobre un usuario cuando visita un sitio web. Algunos sitios utilizan cookies para mantener las sesiones de los usuarios.

De propiedad corporativa, solo para empresas (COBO): modelo de aprovisionamiento de dispositivos móviles empresariales donde el equipo es propiedad de la organización y se prohibe el uso personal.

De propiedad corporativa, habilitado personalmente (COPE): modelo de aprovisionamiento de dispositivos móviles empresariales donde el dispositivo sigue siendo propiedad de la organización, pero se permite determinado uso personal, como el correo electrónico privado, las redes sociales y la navegación web.

Control correctivo: tipo de control de seguridad que actúa luego de incidente para eliminar o minimizar su impacto.

correlation: función de análisis de registros que vincula datos de registro y de estado para identificar un patrón que debería registrarse o alertarse como un evento.

Canal encubierto: tipo de ataque que subvierte los sistemas y políticas de seguridad de la red para transferir datos sin autorización o detección.

Recolección de credenciales: técnicas de ingeniería social para recopilar credenciales válidas que se utilizarán para obtener un acceso no autorizado.

Reproducción de credenciales: ataque que utiliza un token de autenticación capturado para iniciar una sesión no autorizada sin necesidad de descubrir la contraseña en texto plano de una cuenta.

Escaneo acreditado: exploración que utiliza credenciales, como nombres de usuario y contraseñas, para realizar un análisis exhaustivo durante el escaneo de vulnerabilidades, lo que proporcionará más información mientras se audita la red.

Tasa de error de cruce (CER): factor de evaluación biométrica que expresa el punto en el que se encuentran la tasa de error de aceptación (FAR) y la tasa de error de rechazo (FRR), siendo un valor bajo indicativo de un mejor rendimiento.

Falsificación de solicitud entre sitios (CSRF): script malicioso alojado en el sitio del atacante que puede explotar una sesión iniciada en otro sitio del mismo navegador.

Secuencias de comandos entre sitios (XSS): script malicioso alojado en el sitio del atacante o codificado en un enlace inyectado en un sitio de confianza diseñado para comprometer a los clientes que navegan por el sitio de confianza, burlando el modelo de seguridad de las zonas de confianza del navegador.

Criptoanálisis: ciencia, arte y práctica de descifrar códigos y cifrados.

Primitiva criptográfica: una sola función hash, cifrado simétrico o cifrado asimétrico.

Criptografía: ciencia y práctica de alterar datos para hacerlos ininteligibles para partes no autorizadas.

Cryptominer (criptominero): malware que secuestra recursos del equipo para crear criptomonedas.

Inteligencia de ciberamenazas (CTI): proceso de investigación, recopilación, análisis y difusión de información sobre amenazas emergentes y fuentes de amenazas.

Marco de ciberseguridad (CSF): estándares, mejores prácticas y pautas para una gestión eficaz de los riesgos de seguridad. Algunos marcos son generales, mientras que otros son específicos de la industria o los tipos de tecnología.

Dark web (web oscura): recursos en Internet distribuidos entre nodos anónimos y protegidos del acceso general a través de múltiples capas de cifrado y enrutamiento.

Panel: consola que presenta la información seleccionada en un formato de fácil comprensión, como una visualización.

Adquisición de datos: en el análisis forense digital, el método y las herramientas utilizados para crear una copia forense de los datos de un dispositivo de origen, como la memoria del sistema o un disco duro.

Datos en reposo: información que se almacena principalmente en medios específicos, en lugar de transferirse de un medio a otro.

Brecha de datos: cuando los datos confidenciales o privados se leen, copian o modifican sin autorización. Los eventos de brechas de datos pueden estar sujetos a requisitos de notificación e informes.

Clasificación de datos: proceso de aplicación de etiquetas de confidencialidad y privacidad a la información.

Controlador de datos: en las regulaciones de privacidad, la entidad que determina cómo y por qué se recopilan, almacenan y utilizan los datos personales.

Custodio de datos: individuo responsable de administrar el sistema en el que se almacenan los activos de datos, incluso la responsabilidad de aplicar el control de acceso, el cifrado y las medidas de respaldo/recuperación.

Exfiltración de datos: proceso mediante el cual un atacante toma datos almacenados dentro de una red privada y los traslada a una red externa.

Exposición de datos: vulnerabilidad de software donde un atacante puede eludir los controles de acceso y recuperar datos confidenciales o sensibles del sistema de archivos o la base de datos.

Historiador de datos: software que agrega y cataloga datos de múltiples fuentes dentro de un sistema de control industrial.

Datos en tránsito: información que se transmite entre dos hosts, como a través de una red privada o Internet.

Datos en uso: información que está presente en la memoria volátil de un host, como la memoria del sistema o la caché.

Inventario de datos: lista de datos clasificados o información almacenada o procesada por un sistema.

Prevención de pérdida de datos (DLP): solución de software que detecta y previene que la información sensible se almacene en sistemas no autorizados o se transmita por redes no autorizadas.

Enmascaramiento de datos: un método de desidentificación en el que los datos reales se sustituyen por etiquetas genéricas o de marcador de posición, al tiempo que se conserva la estructura o el formato de los datos originales.

Propietario de datos: una función sénior (ejecutivo) con la responsabilidad final de mantener la confidencialidad, integridad y disponibilidad de un activo de información.

Plano de datos: funciones que hacen cumplir las decisiones de políticas configuradas en el plano de control y facilitan las transferencias de datos.

Procesador de datos: en la normativa de privacidad, una entidad de confianza con una copia de los datos personales para realizar el almacenamiento o análisis en nombre del recolector de datos.

Retención de datos: proceso que usa una organización para mantener la existencia y el control sobre ciertos datos con el fin de cumplir con las políticas comerciales o las leyes y regulaciones aplicables.

Titular de los datos: una persona que se identifica por los datos de privacidad.

Cifrado de bases de datos: aplicación de cifrado a nivel de tabla, campo o registro a través de un sistema de administración de bases de datos en lugar de a través del sistema de archivos.

Comando dd: comando de Linux que realiza una copia bit a bit de un archivo de entrada, por lo general utilizado para imágenes de disco.

Arquitectura informática descentralizada: un modelo en el que el procesamiento y el almacenamiento de datos se distribuyen en múltiples ubicaciones o dispositivos.

Engaño e interrupción: herramientas y técnicas de resiliencia de ciberseguridad para aumentar el costo de la planificación de ataques para el actor de amenazas.

Desduplicación: una técnica para eliminar copias duplicadas de datos repetidos. En SIEM, la eliminación de la información redundante proporcionada por varios sistemas monitoreados.

Defensa en profundidad: estrategia de seguridad que ubica las capas de diversas categorías y funciones de control de seguridad en lugar de basarse en controles perimetrales.

Prueba de penetración defensiva: el equipo defensivo en una prueba de penetración o ejercicio de respuesta a incidentes.

Ataque de denegación de servicio (DoS): cualquier tipo de ataque físico, de aplicación o de red que afecta a la disponibilidad de un recurso administrado.

Dependencias: recursos y otros servicios que deben estar disponibles y en ejecución para que se inicie un servicio.

Desaprovisionamiento: el proceso de eliminación de una cuenta, host o aplicación del entorno de producción. Esto requiere la revocación de cualquier acceso privilegiado que se haya asignado al objeto.

Destrucción: una técnica de eliminación de activos que garantiza que los restos de datos se vuelvan físicamente inaccesibles e irrevocables mediante desmagnetización, trituración o incineración.

Detectabilidad: parámetro de evaluación de riesgos que define la probabilidad de que una empresa detecte la ocurrencia de un riesgo antes de que afecte el proyecto, el proceso o el usuario final.

Detección: un proceso de respuesta a incidentes que correlaciona datos de eventos para determinar si son indicadores de un incidente.

Control detectivo: tipo de control de seguridad que actúa durante un incidente para identificar o registrar que está ocurriendo.

Ubicación de dispositivos: consideraciones para posicionar los controles de seguridad a fin de proteger las zonas de la red y los hosts individuales para implementar una estrategia de defensa profunda y cumplir con los objetivos generales de seguridad.

DevSecOps: una combinación de desarrollo de software, operaciones de seguridad y operaciones de sistemas, que hace referencia a la práctica de integrar cada disciplina con las demás.

Ataque de diccionario: tipo de ataque de contraseña que compara las contraseñas cifradas con una lista predeterminada de posibles valores de contraseña.

Diffie-Hellman (DH): técnica criptográfica que proporciona un intercambio seguro de claves.

Certificado digital: información de identificación y autenticación presentada en el formato X.509 y emitida por una autoridad certificadora (CA) como garantía de que un par de claves (identificado por la clave pública incluida en el certificado) es válido para un sujeto concreto (usuario o host).

Firma digital: síntesis del mensaje cifrado usando la clave privada del remitente que se anexa a un mensaje para autenticar al remitente y probar la integridad del mensaje.

Control directivo: un tipo de control que hace cumplir una regla de comportamiento a través de una política o contrato.

Servicio de directorio: un servicio de red que almacena información de identidad sobre todos los objetos de una red específica, que incluye usuarios, grupos, servidores, computadoras del cliente e impresoras.

Salto de directorios: ataque a una aplicación que permite acceder a comandos, archivos y directorios que pueden o no estar conectados al directorio raíz del documento web.

Ataque de disociación: tramas de suplantación de identidad para desconectar una estación inalámbrica y tratar de obtener datos de autenticación para descifrar.

Recuperación ante desastres (DR): plan documentado y con recursos que muestra acciones y responsabilidades que se utilizarán en respuesta a incidentes críticos.

Control de acceso discrecional (DAC): un modelo de control de acceso en el que cada recurso está protegido por una lista de control de acceso (ACL) administrada por el propietario (o propietarios) del recurso.

Desinformación: tipo de ataque que falsifica un recurso de información en el que por lo general confían otros.

Enajenación/desmantelamiento: en la gestión de activos, las políticas y los procedimientos que rigen la eliminación de dispositivos y software de las redes de producción, y su posterior eliminación a través de la venta, donación o como desperdicio.

Nombre distinguido (DN): una colección de atributos que definen un identificador único para cualquier recurso dado dentro de un directorio similar a X.500.

Denegación de servicio distribuido (DDoS): ataque que implica el uso de computadoras y dispositivos infectados conectados a Internet para interrumpir el flujo normal de tráfico de un servidor o servicio abrumando el objetivo con tráfico.

DoS reflejado distribuido (DRDoS): se crea una solicitud maliciosa a un servidor legítimo y se envía como un enlace a la víctima, de modo que una falla del lado del servidor hace que el componente malicioso se ejecute en el navegador del objetivo.

Envenenamiento del DNS: ataque en el que un actor de amenaza inyecta falsos registros de recursos en la caché de un cliente o servidor para redirigir un nombre de dominio a una dirección IP de su elección.

Sumidero de DNS: un registro DNS temporal que redirige el tráfico malicioso a una dirección IP controlada.

Modelo de objetos de documento (DOM): cuando los atacantes envían scripts maliciosos a la implementación de JavaScript del lado del cliente de una aplicación web para ejecutar su ataque únicamente en el cliente.

Extensiones de seguridad del sistema de nombres de dominio (DNSSEC): protocolo de seguridad que proporciona autenticación de datos DNS y garantiza la integridad de los datos DNS.

Autenticación de mensajes, informes y conformidad basada en dominios [DMARC]: marco para garantizar la correcta aplicación del marco de directivas de remitente (SPF) y el correo identificado por claves de dominio (DKIM), utilizando una política publicada como registro del sistema de nombres de dominio (DNS).

DomainKeys Identified Mail (DKIM): mecanismo de autenticación criptográfica para correo electrónico que utiliza una clave pública publicada como un registro DNS.

Ataque de degradación: un ataque criptográfico en el que el atacante aprovecha la necesidad de compatibilidad con versiones anteriores para obligar a un sistema informático a abandonar el uso de mensajes cifrados en favor de los mensajes de texto sin formato.

Diligencia debida: un principio legal que establece que un sujeto utilizó las mejores prácticas o las precauciones razonables al configurar, configurar y mantener un sistema.

Debido proceso: el debido proceso es un término utilizado en el derecho consuetudinario de los Estados Unidos y el Reino Unido para exigir que las personas solo sean condenadas por delitos después de la aplicación justa de las leyes del país.

Archivo de volcado: un archivo que contiene datos obtenidos de la memoria del sistema.

Análisis dinámico: pruebas de software que examinan el comportamiento del código durante el tiempo de ejecución. Ayuda a identificar posibles problemas de seguridad, de rendimiento y otros.

Descubrimiento electrónico: procedimientos y herramientas para recopilar, conservar y analizar evidencia digital.

Sistema integrado: sistema electrónico diseñado para realizar una función específica y dedicada, como un microcontrolador en un tratamiento intravenoso o componentes en un sistema de control que administra una planta de tratamiento de agua.

Carga de seguridad encapsuladora (ESP): subprotocolo de IPSec que permite el cifrado y la autenticación de la cabecera y la carga útil de un paquete de datos.

Cifrado: codificar los caracteres utilizados en un mensaje para que el mensaje pueda verse, pero no entenderse ni modificarse, a menos que se pueda descifrar. El cifrado proporciona un medio seguro para transmitir datos y autenticar a los usuarios. También se utiliza para almacenar datos de forma segura. El cifrado utiliza diferentes tipos de encriptación y una o más claves. El tamaño de la clave es un factor que determina la seguridad del producto de cifrado.

Nivel de cifrado: objetivo para el cifrado de datos en reposo, que va desde más granular (archivo o fila/registro) a menos granular (volumen/partición/disco o base de datos).

Detección y respuesta de puntos de conexión (EDR): agente de software que recopila datos y registros del sistema para su análisis por parte de un sistema de monitoreo con el fin de proporcionar una detección temprana de amenazas.

Registro de punto de conexión: un objetivo para eventos relacionados con la seguridad y generados por agentes de detección de intrusiones y malware basados en host.

Autenticación empresarial: modo de autenticación de la red inalámbrica en el que el punto de acceso actúa como paso para las credenciales que son verificadas por un servidor AAA.

Gestión de riesgos empresariales (ERM): el proceso integral de evaluación, medición y mitigación de los numerosos riesgos que impregnan una organización.

Ataque ambiental: una amenaza física dirigida contra los sistemas de energía, refrigeración o extinción de incendios.

Variables ambientales: en la evaluación de vulnerabilidades, factores o métricas debido a la configuración local de la red o del host que aumentan o disminuyen la probabilidad base y el nivel de riesgo de impacto.

Efímera: en criptografía, una clave que se usa solo dentro del contexto de una sola sesión.

Erradicación: un proceso de respuesta ante incidentes donde se eliminan herramientas y configuraciones maliciosas en hosts y redes.

Escalamiento: en el contexto de los procedimientos de soporte, la respuesta a incidentes y la creación de informes de vulneraciones, el escalamiento es el proceso de involucrar a personal experto y sénior para ayudar en la gestión de problemas.

Custodia: en la gestión de claves, el almacenamiento de una clave de respaldo con un tercero.

Visor de eventos: una consola de Windows relacionada con la visualización y exportación de eventos en el formato de archivo de registro de Windows.

Punto de acceso falso: punto de acceso inalámbrico que engaña a los usuarios haciéndoles creer que es un punto de acceso a la red legítimo.

Gestión de excepciones: una vulnerabilidad de la aplicación que se define por cómo responde una aplicación a errores inesperados que pueden provocar agujeros en la seguridad de una aplicación.

factor de exposición (EF): en el cálculo de riesgos, el porcentaje del valor de un activo que se perdería durante un incidente de seguridad o escenario de desastre.

Protocolo de autenticación extensible (EAP): marco para negociar métodos de autenticación que permite a los sistemas utilizar identificadores basados en hardware, como escáneres de huella digital o lectores de tarjetas inteligentes, para la autenticación, y establecer túneles seguros a través de los cuales presentar las credenciales.

Protocolo de autenticación extensible a través de LAN (EAPoL): un mecanismo de control de acceso a la red (PNAC) basado en puertos que permite el uso de la autenticación EAP cuando un host se conecta a un switch Ethernet.

Lenguaje de marcado extensible (XML): sistema para estructurar documentos de modo que sean leíbles por personas y por máquinas. La información del documento se coloca dentro de etiquetas, que describen cómo se estructura la información en el documento.

Extorsión: exigir un pago para evitar o detener algún tipo de ataque.

Factores: en el diseño de autenticación, diferentes tecnologías para implementar la autenticación, como el conocimiento, la propiedad o token y la biométrica o inherencia. Estos se distinguen como lo que una persona conoce, tiene o es.

Cierre automático ante fallas: una configuración de control de seguridad que bloquea el acceso a un recurso en caso de falla.

Apertura automática ante fallas: una configuración de control de seguridad que garantiza el acceso continuo al recurso en caso de falla.

Conmutación por error: una técnica que garantiza que un componente, dispositivo o aplicación redundante pueda asumir de manera rápida y eficiente la funcionalidad de un activo que falló.

Telemetría falsa: estrategia de engaño que devuelve datos falsificados en respuesta a sondeos de red.

Tasa de falsa aceptación (FAR): una métrica de evaluación biométrica que mide el número de usuarios no autorizados a los que se les permite el acceso por error.

Falso negativo: en el escaneo de seguridad, un caso que no se informa cuando debería.

Falso positivo: en el escaneo de seguridad, un caso que se informa cuando no debería.

Tasa de falso rechazo (FRR): una métrica de evaluación biométrica que mide el número de sujetos válidos a los que se les niega el acceso.

Tolerancia a fallas: protección contra fallas del sistema al proporcionar capacidad adicional (redundante). En general, los sistemas que resisten las fallas identifican y eliminan los puntos únicos de falla.

Federación: proceso que proporciona una capacidad de inicio de sesión compartida entre varios sistemas y empresas. Básicamente, conecta los servicios de gestión de identidades de varios sistemas.

Fencing (Rejas): barrera de seguridad diseñada para evitar el acceso no autorizado al perímetro de un sitio.

Monitoreo de la integridad de archivos (FIM): tipo de software que revisa los archivos del sistema para asegurarse de que no hayan sido manipulados.

Protocolo de transferencia de archivos (FTP): protocolo de aplicación utilizado para transferir archivos entre hosts de la red. Las variantes incluyen S(ecure) FTP, FTP con SSL (FTPS y FTPES) y T(rivial) FTP. El FTP utiliza los puertos 20 y 21.

Datos financieros: datos sobre cuentas bancarias y de inversión, además de información como nóminas y declaraciones de impuestos.

Registro del cortafuegos: un objetivo para los datos de eventos relacionados con las reglas de acceso configuradas para el registro.

Servicios de primera respuesta: la primera persona o equipo con experiencia en llegar a la escena de un incidente.

Análisis forense: proceso de recopilación y presentación de evidencias informáticas para un juicio. La evidencia digital es latente, es decir, debe interpretarse. Esto significa que se debe tener mucho cuidado para demostrar que la evidencia no se manipuló ni se falsificó.

Ataque de falsificación: un ataque que aprovecha una autenticación débil para realizar una solicitud a través de una sesión secuestrada.

Fraude: falsificar registros, como en el caso de un fraude interno que implica la manipulación de cuentas.

FTPS: un tipo de FTP que utiliza TLS para la confidencialidad.

Cifrado de disco completo (FDE): el cifrado de todos los datos en un disco (incluyendo los archivos del sistema, archivos temporales y el archivo de paginación) se puede realizar a través de un sistema operativo compatible, software de terceros o en el nivel del controlador por medio del mismo dispositivo de disco.

Análisis de deficiencias: un análisis que mide la diferencia entre los estados actual y deseado con el fin de ayudar a evaluar el alcance del trabajo incluido en un proyecto.

Geofencing (geovallado): control de seguridad que puede imponer una frontera virtual basada en la geografía del mundo real.

Dispersión geográfica: un mecanismo de resiliencia donde los recursos de almacenamiento y procesamiento de datos se replican entre sitios físicamente distantes.

Geolocalización: la identificación o estimación de la ubicación física de un objeto, como una fuente de radar, un teléfono móvil o un dispositivo informático conectado a Internet.

Sistema de posicionamiento global (GPS): medios para determinar la posición de un receptor en la Tierra según la información recibida de satélites orbitales.

Gobernanza: crear y monitorear políticas y procedimientos efectivos para administrar activos, como datos, y garantizar el cumplimiento de las regulaciones de la industria y la legislación local, nacional y global.

Junta directiva: ejecutivos sénior y partes interesadas externas con la responsabilidad de establecer la estrategia y garantizar el cumplimiento.

Comité de gobernanza: líderes y expertos en la materia con la responsabilidad de definir políticas, procedimientos y estándares dentro de un dominio o alcance específico.

Cuenta de grupo: una cuenta de grupo es una recopilación de cuentas de usuarios que es útil al establecer permisos para archivos y derechos para los usuarios, porque cuando muchas personas necesitan el mismo nivel de acceso, se podría establecer un grupo que incluya a todos los usuarios relevantes.

Objetos de directiva de grupo (GPO): en un dominio de Windows, una forma de implementar configuraciones por usuario y por computadora, como una política de contraseñas, restricciones de cuenta, estado del cortafuegos, etc.

Directrices: recomendaciones de mejores prácticas y consejos para elementos de configuración donde las políticas y los estándares detallados y estrictamente aplicables no son prácticos.

Hacker: los hackers éticos, término que se utiliza a menudo para referirse a alguien que irrumpe en sistemas informáticos o propagan virus, prefieren considerarse expertos y exploradores de sistemas de seguridad informática.

Hacktivista: actor de amenaza motivado por un problema social o una causa política.

Token de autenticación duro o de hardware: un token de autenticación generado por un criptoprocesador en un dispositivo de hardware dedicado. Como el token nunca se transmite directamente, esto implementa un factor de propiedad dentro de un esquema de autenticación multifactor.

Endurecimiento: proceso que consiste en hacer segura la configuración de un host o de una aplicación reduciendo su superficie de ataque, mediante la ejecución solo de los servicios necesarios, la instalación de software de monitorización para protegerla contra los programas maliciosos e intrusiones, y el establecimiento de un cronograma de mantenimiento que garantice que el sistema esté actualizado para ser seguro contra las vulnerabilidades del software.

Código de autenticación de mensajes basado en hash (HMAC): un método utilizado para verificar la integridad y autenticidad de un mensaje combinando un hash criptográfico del mensaje con una clave secreta.

Hashing: función que convierte una cadena de entrada de longitud arbitraria en una cadena de salida de longitud fija. Una función hash criptográfica realiza esto de una manera que reduce la posibilidad de colisiones, en la que dos entradas diferentes producen la misma salida.

Ley de Portabilidad y Responsabilidad de Seguros Médicos (HIPAA): Ley Federal de EE. UU. que protege el almacenamiento, la lectura, la modificación y la transmisión de datos personales relacionados con la atención médica.

Mapa de calor: en un sondeo del sitio Wi-Fi, un diagrama que muestra la intensidad de la señal y la utilización del canal en diferentes lugares.

Matriz de riesgo de mapa de calor: una tabla gráfica que indica la probabilidad y el impacto de los factores de riesgo identificados para un flujo de trabajo, proyecto o departamento que se utiliza como referencia para las partes interesadas.

Heurística: un método que utiliza comparaciones y semejanzas de características en lugar de coincidencias de firmas específicas para identificar si el objetivo de la observación es malicioso.

Alta disponibilidad (HA): métrica que define qué tanto se acercan los sistemas al objetivo de proporcionar disponibilidad de datos el 100 % del tiempo mientras se mantiene un nivel alto de rendimiento del sistema.

Honeypot (señuelo): host (honeypot), red (honeynet), archivo (honeyfile) o credencial/token (honeytoken) configurado con el propósito de alejar a los atacantes de los activos de valor real o descubrir estrategias de ataque y debilidades en la configuración de seguridad.

Escalamiento horizontal de privilegios: cuando un usuario accede a recursos específicos a los que no tiene derecho o los modifica.

Cortafuegos basado en host: una aplicación de software que se ejecuta en un solo host y está diseñada para proteger solo a ese host.

Sistema de detección de intrusiones basado en host (HIDS): un tipo de IDS que monitorea un sistema informático en busca de comportamientos inesperados o cambios drásticos en el estado del sistema.

Sistema de prevención de intrusiones basado en host (HIPS): protección de punto de conexión que permite detectar y prevenir actividades maliciosas a través de la firma y la coincidencia de patrones heurísticos.

Sitio caliente: sitio de procesamiento alternativo totalmente configurado que puede ponerse en línea al instante o muy rápidamente después de un desastre.

VPN HTML5: uso de las características de HTML5 para implementar conexiones remotas de desktop/VPN a través del software del navegador (sin clientes).

Interfaz humano-máquina (HMI): controles de entrada y salida de un controlador lógico programable (PLC) que permiten al usuario configurar y monitorear el sistema.

Datos legibles por humanos: información almacenada en un tipo de archivo al que los seres humanos pueden acceder y comprender utilizando un software de visualización básico, como documentos, imágenes, video y audio.

Nube híbrida: implementación en la nube que utiliza elementos tanto privados como públicos.

Ataque híbrido de contraseña : ataque en el que se utilizan varios métodos de ataque, que incluyen ataques de diccionario, tabla de arco iris y fuerza bruta al intentar descifrar una contraseña.

Identificación: proceso mediante el cual se emite una cuenta de usuario (y sus credenciales) a la persona correcta. A veces se denomina registro.

Gestión de identidad y acceso (IAM): proceso de seguridad que proporciona mecanismos de identificación, autenticación y autorización para que los usuarios, las computadoras y otras entidades trabajen con activos de la organización como redes, sistemas operativos y aplicaciones.

Proveedor de identidad: en una red federada, el servicio que mantiene la cuenta del usuario y realiza la autenticación.

Registro IDS/IPS: un objetivo para los datos de eventos relacionados con las reglas de detección/prevención configuradas para el registro.

IEEE 802.1X: un estándar para encapsular comunicaciones EAP a través de una LAN (EAPoL) o WLAN (EAPoW) para implementar la autenticación basada en puertos.

Impacto: la gravedad del riesgo si se materializa por factores como el alcance, el valor del activo o los impactos financieros del evento.

**Suplantación**: ataque de ingeniería social en el que un atacante se hace pasar por alguien que no es.

Denegación implícita: principio básico de seguridad que establece que a menos que a algo se le haya otorgado acceso explícitamente, se le debe negar el acceso.

Viaje imposible: un indicador potencial de actividad maliciosa donde se realizan intentos de autenticación desde diferentes ubicaciones geográficas en un corto período de tiempo.

Incidente: un evento que interrumpe las operaciones estándar o compromete la política de seguridad.

Ciclo de vida de respuesta a incidentes: procedimientos y pautas que abarcan prioridades, acciones y responsabilidades apropiadas en caso de incidentes de seguridad, divididos en etapas de preparación, detección, análisis, contención, erradicación/recuperación y lecciones aprendidas.

Plan de respuesta a incidentes (IRP): procedimientos específicos que deben llevarse a cabo si se detecta o notifica un determinado tipo de evento.

Indicador de compromiso (IoC): señal de que un activo o red fue atacado o está siendo atacado.

Sistema de posicionamiento interior [IPS]: tecnología que permite obtener la ubicación de un dispositivo en interiores mediante la triangulación de su proximidad a fuentes de radio, como balizas Bluetooth o puntos de acceso Wi-Fi.

Camuflaje industrial: métodos para disfrazar la naturaleza y el propósito de las instalaciones o partes de las instalaciones.

Sistema de control industrial (ICS): red que gestiona los dispositivos integrados (sistemas informáticos que están diseñados para realizar una función dedicada específica).

Políticas de seguridad de la información: un documento o serie de documentos respaldados por la alta gerencia y que detallan los requisitos para proteger los activos de tecnología e información contra amenazas y uso indebido.

Centro de Análisis e Intercambio de Información [ISAC]: grupo sin fines de lucro creado para compartir entre sus miembros información sobre amenazas específicas del sector y buenas prácticas de seguridad.

Organización de intercambio de información: grupos de colaboración que intercambian datos sobre amenazas y vulnerabilidades emergentes de ciberseguridad.

Infraestructura como un Servicio (IaaS): modelo de servicio en la nube que suministra máquinas virtuales e infraestructura de red.

Infraestructura como código (IaC): arquitectura de aprovisionamiento en la que el despliegue de recursos se realiza mediante automatización y orquestación de secuencias de comandos.

Riesgo inherente: riesgo que representará un evento si no se implementan controles para mitigarlo.

Ataque de inyección: un ataque que aprovecha el manejo deficiente de solicitudes o de validación de entradas para ejecutar código arbitrario en el navegador de un cliente o en un servidor.

Inline (en línea): la colocación y configuración de un control de seguridad de red de manera que forme parte del recorrido del cable.

Validación de entradas: cualquier técnica que se utiliza para garantizar que los datos que se introducen en un campo o variable de una aplicación se tratan en forma adecuada en dicha aplicación.

Pruebas de penetración integradas: un enfoque integral que combina diferentes tipos de metodologías y técnicas de prueba de penetración para evaluar las operaciones de seguridad de una organización.

Integridad: el objetivo de seguridad fundamental de mantener la información de la organización precisa, libre de errores y sin modificaciones no autorizadas.

Fusión de inteligencia: en la búsqueda de amenazas, utilizar fuentes de datos de inteligencia de amenazas para automatizar la detección de IoC y TTP del adversario.

Amenaza intencional: un actor de amenaza con un propósito malicioso.

Amenaza interna: tipo de actor de amenaza al que se le asignan privilegios en el sistema y causa un incidente intencionado o no.

Interno/externo: el grado de acceso que posee un actor de amenazas antes de iniciar un ataque. Un actor de amenazas externo no tiene privilegios permanentes, mientras que a un actor interno se le concedieron algunos permisos de acceso.

Encabezado de Internet: un registro de los servidores de correo electrónico involucrados en la transferencia de un mensaje de correo electrónico de un remitente a un destinatario.

Intercambio de claves de Internet [IKE]: marco para crear una Asociación de Seguridad (SA) utilizada con IPSec. Una SA establece que dos hosts confían entre sí (se autentican) y acuerdan protocolos seguros y conjuntos de cifrado que utilizarán para intercambiar datos.

Protocolo de acceso a mensajes de Internet [IMAP]: protocolo de aplicación que proporciona un medio para que un cliente acceda y administre los mensajes de correo electrónico almacenados en un buzón en un servidor remoto. IMAP4 utiliza el número de puerto TCP 143, mientras que la versión segura IMAPS utiliza TCP/993.

Internet de las cosas (IoT): dispositivos que pueden informar sobre el estado y los datos de configuración y ser gestionados a distancia a través de redes IP.

Internet Protocol (IP) (Protocolo de Internet [IP]): el protocolo de capa de red (Internet) en la suite TCP/IP que proporciona direccionamiento de paquetes y enrutamiento para todos los protocolos de nivel superior en el conjunto de aplicaciones.

Seguridad del Protocolo de Internet [IPSec]: conjunto de protocolos de red que se utiliza para asegurar los datos mediante la autenticación y el cifrado a medida que los datos viajan a través de la red o de Internet.

Internet Relay Chat (IRC): grupo de protocolos de comunicación que permite a los usuarios chatear, enviar mensajes privados y compartir archivos.

Sistema de detección de intrusión [IDS]: dispositivo o software de seguridad que analiza los datos de un analizador de paquetes para identificar el tráfico que infringe las políticas o reglas.

Sistema de prevención de intrusión (IPS): dispositivo o software de seguridad que combina capacidades de detección con funciones que pueden bloquear activamente los ataques.

Exportación de información de flujo IP (IPFIX): versión basada en estándares del marco Netflow.

Aislamiento: eliminar o restringir severamente las vías de comunicación a un dispositivo o sistema en particular.

Biblioteca de infraestructura de TI (ITIL): un marco de mejores prácticas de TI, que enfatiza la alineación de la Gestión de servicios de TI (ITSM) con las necesidades comerciales. El gobierno del Reino Unido desarrolló la ITIL por primera vez en 1989. La ITIL 4 se lanzó en 2019 y ahora AXELOS la comercializa.

Jailbreaking (fuga de la cárcel): elimina el sello de protección y cualquier restricción específica del sistema operativo para dar a los usuarios un mayor control sobre el dispositivo.

Notación de objetos en JavaScript [JSON]: formato de archivo que utiliza pares de valor-atributo para definir configuraciones en una estructura de lectura y consumo fácil, tanto para los humanos como para las máquinas.

Registro por diario: un método utilizado por los sistemas de archivos para registrar cambios que aún no se realizaron en el sistema de archivos en un objeto llamado diario.

Servidor Jump: un servidor reforzado que proporciona acceso a otros hosts.

Kerberos: servicio de autenticación y autorización de inicio de sesión único que se basa en un sistema de concesión de vales por tiempo.

centro de distribución de claves (KDC): un componente de Kerberos que autentica a los usuarios y emite tickets (tokens).

clave de cifrado de clave [KEK]: en el cifrado de almacenamiento, la clave privada que se utiliza para cifrar la clave de cifrado de medios masivos (MEK) simétrica. Esto significa que un usuario debe autenticarse para descifrar la MEK y acceder a los medios.

intercambio de claves: cualquier método por medio del cual las claves criptográficas se transfieren entre usuarios, lo que permite el uso de un algoritmo criptográfico.

longitud de la clave: tamaño de una clave criptográfica en bits. Las claves más largas generalmente ofrecen mayor seguridad, pero las longitudes de clave para diferentes cifrados no son directamente comparables.

sistema de gestión de claves: en PKI, los procedimientos y las herramientas que centralizan la generación y el almacenamiento de claves criptográficas.

indicador de riesgo de claves (KRI): el método mediante el que se identifican y analizan los riesgos emergentes para que se puedan adoptar cambios y así evitar de manera proactiva problemas potenciales.

estiramiento de claves: una técnica que refuerza la entrada potencialmente débil para la generación de claves criptográficas, como contraseñas o frases de contraseña creadas por personas, contra ataques de fuerza bruta.

registrador de pulsaciones de teclas: registrador de pulsaciones de teclas Software o hardware malintencionado que puede registrar las pulsaciones de teclas del usuario.

cadena de eliminación: un modelo desarrollado por Lockheed Martin que describe las etapas por las que un actor de amenaza progresa hacia una intrusión en la red.

movimiento lateral: proceso mediante el cual un atacante es capaz de desplazarse de una parte a otra en un entorno informático.

cortafuegos de capa 4: un cortafuegos de inspección de estado que puede monitorear las sesiones TCP y el tráfico UDP.

cortafuegos de capa 7: un cortafuegos de inspección de estado que puede filtrar el tráfico en función de encabezados y datos específicos del protocolo de aplicación, como datos web o de correo electrónico.

privilegio mínimo: un principio básico de seguridad que establece que a algo se le debe asignar los derechos, privilegios o información mínimos necesarios para desempeñar su función.

datos legales: documentos y registros que se relacionan con asuntos legales, como contratos, propiedades, casos judiciales y presentaciones regulatorias.

retención legal: un proceso diseñado para preservar toda la información relevante cuando se espera que se produzca un litigio.

informe de lecciones aprendidas (LLR): un análisis de eventos que pueden aportar ideas sobre cómo mejorar los procesos de respuesta y apoyo en el futuro.

nivel de sofisticación/capacidad: una clasificación formal de los recursos y la experiencia disponibles para un actor de amenazas.

iluminación: mecanismos de seguridad física que garantizan que un sitio esté suficientemente iluminado para que los empleados y los invitados se sientan seguros y para que los sistemas de vigilancia basados en cámaras funcionen bien.

Protocolo ligero de acceso a directorios [LDAP]: un protocolo usado para acceder a las bases de datos del directorio de red que almacenan información sobre usuarios autorizados y sus privilegios, así como otro tipo de información organizacional.

Protocolo ligero y seguro de acceso a directorios [LDAP Secure]: un método para implementar LDAP utilizando cifrado SSL/TLS.

probabilidad: en el análisis cualitativo de riesgos, la probabilidad de un evento que se expresa como una escala determinada de manera subjetiva, como alta o baja.

oyente/recopilador: un dispositivo de red que recopila o recibe datos de registro o estado de otros sistemas de red.

balanceador de carga: tipo de switch, enrutador o software que distribuye las solicitudes de los clientes entre diferentes recursos, como enlaces de comunicaciones o servidores configurados de manera similar. Esto proporciona tolerancia a errores y permite mejorar el rendimiento.

agregación de registros: análisis de información de múltiples fuentes de datos de registros y eventos de seguridad que se puede presentar en un formato coherente con opción de búsqueda.

datos de registro: el software del sistema operativo y las aplicaciones se pueden configurar para registrar eventos automáticamente. Esto proporciona información valiosa para la resolución de problemas. Los registros de seguridad proporcionan un registro de auditoría de las acciones realizadas en el sistema, así como advertencias sobre actividades sospechosas. Es importante que la configuración y los archivos de registro sean a prueba de manipulaciones.

bomba lógica: un programa o script malicioso que se ejecuta en circunstancias particulares o en respuesta a un evento definido.

segmentación lógica: topología de red impuesta por la configuración del switch, el enrutador y el cortafuegos, donde se impide o restringe la comunicación de los hosts de un segmento de la red con los hosts de otros segmentos.

señuelo: un tipo de ataque que motiva a una víctima a usar o abrir un dispositivo extraíble, un documento, una imagen o un programa que oculte malware.

aprendizaje automático [ML]: un componente de IA que permite a una máquina desarrollar estrategias para resolver una tarea dado un conjunto de datos etiquetado donde las características se identificaron de forma manual, pero sin más instrucciones explícitas.

proceso malicioso: un proceso ejecutado sin la debida autorización del propietario del sistema con el fin de dañar o comprometer dicho sistema.

actualización maliciosa: una vulnerabilidad en el repositorio de software o la cadena de suministro que un actor de amenazas puede explotar para agregar código malicioso a un paquete.

programa malicioso: software que tiene un propósito malicioso, por lo general instalado sin el consentimiento (o conocimiento) del usuario.

Control de acceso obligatorio [MAC]: un modelo de control de acceso en el que los recursos están protegidos por reglas inflexibles definidas por el sistema. Los recursos (objetos) y usuarios (sujetos) a los que se les asigna un nivel de autorización (o etiqueta).

maniobra: en la búsqueda de amenazas, el concepto de que el actor y el defensor de la amenaza pueden usar estrategias de engaño o contraataque para obtener una ventaja.

acuerdo maestro de servicios [MSA]: un contrato que establece la precedencia y las directrices para cualquier documento de negocios que se ejecute entre dos partes.

tiempo de inactividad máximo tolerable (MTD): período más largo en el que un proceso puede ser inoperable sin causar una falla empresarial irreversible.

tiempo medio entre fallas [MTBF]: métrica para un dispositivo o componente que predice el tiempo esperado entre fallas.

tiempo medio de reparación/reemplazo/recuperación [MTTR]: métrica que representa el tiempo medio que tarda un dispositivo o componente en ser reparado, sustituido o recuperado de otro modo tras una falla.

filtrado de dirección de Acceso de medios (filtrado de MAC): aplicar una lista de control de acceso a un conmutador o punto de acceso para que solo los clientes con direcciones MAC aprobadas se puedan conectar a él.

Memorando de acuerdo (MoA):documento legal que conforma la base para que dos partes cooperen sin un contrato formal (un acuerdo de cooperación). Los organismos públicos suelen utilizar los MoA.

memorando de entendimiento (MoU): suele ser un acuerdo preliminar o exploratorio para expresar la intención de trabajar juntos que no es jurídicamente vinculante y no implica el intercambio de dinero.

inyección de memoria: una vulnerabilidad que un actor de amenazas puede explotar para ejecutar código malicioso con el mismo nivel de privilegio que el proceso vulnerable.

Algoritmo de resumen de mensajes v5 (MD5): una función hash criptográfica que produce una salida de 128 bits.

metadatos: información almacenada o registrada como propiedad de un objeto, estado de un sistema o transacción.

microservicio: módulo independiente de una sola función con interfaces y operaciones bien definidas y ligeras. Por lo general, este estilo de arquitectura permite la entrega rápida, frecuente y confiable de aplicaciones complejas.

pérdida de registros: un indicador potencial de actividad maliciosa donde se eliminan o manipulan eventos o archivos de registro.

función esencial de la misión (MEF): actividad empresarial u organizativa que es demasiado crítica como para aplazarla más de unas horas, si es que admite aplazamiento alguno.

administración de dispositivos móviles [MDM]: proceso y tecnologías de soporte para rastrear, controlar y proteger la infraestructura móvil de la organización.

supervisión/seguimiento de activos: software y procesos de enumeración e inventario que garantizan que los activos físicos y de datos cumplan con las líneas de base de configuración y rendimiento, y que no hayan sido manipulados ni hayan sufrido otros accesos no autorizados.

múltiples nubes: un modelo de implementación en la nube en el que el consumidor de la nube utiliza múltiples servicios de nube pública.

autenticación de multifactores [MFA]: esquema de autenticación que requiere que el usuario presente al menos dos factores diferentes como credenciales; por ejemplo, algo que sabes, algo que tienes, algo que eres, algo que haces y en algún lugar en el que estás. Especificar dos factores se conoce como 2FA.

actor estatal nación: tipo de actor de amenaza que cuenta con el apoyo de los recursos de los servicios militares y de seguridad de su país anfitrión.

Instituto Nacional de Estándares y Tecnología (NIST): desarrolla estándares de seguridad informática utilizados por las agencias federales de los EE. UU. y publica guías e investigaciones sobre las mejores prácticas en materia de ciberseguridad.

comunicación de campo cercano [NFC]: un estándar para las comunicaciones de radio de dos vías en distancias muy cortas (alrededor de cuatro pulgadas), que facilita el pago sin contacto y tecnologías similares. NFC se basa en RFID.

NetFlow: medio desarrollado por Cisco para reportar información de flujo de red a una base de datos estructurada. NetFlow permite una mejor comprensión de los flujos de tráfico IP tal y como los utilizan las diferentes aplicaciones de red y los hosts.

control de acceso a la red (NAC): término general para el conjunto de protocolos, políticas y hardware que autentifican y autorizan el acceso a una red a nivel del dispositivo.

ataque de red: un ataque dirigido contra la infraestructura de red cableada o inalámbrica, que incluye reconocimiento, denegación de servicio, recolección de credenciales, en ruta, escalamiento de privilegios y exfiltración de datos.

detección de anomalías y comportamiento de red (NBAD): una herramienta de monitoreo de seguridad que supervisa los paquetes de red en busca de comportamientos anómalos basados en firmas conocidas.

virtualización de la función de la red (NFV): aprovisionamiento de dispositivos de red virtuales, como switches, enrutadores y cortafuegos, a través de VM y contenedores.

registro de red: un destino para eventos de acceso y sistema generados por un dispositivo de red, como un switch, un punto de acceso inalámbrico o un enrutador.

monitoreo de red: software de auditoría que recolecta información del estado y la configuración de los dispositivos de red. Muchos productos se basan en el Protocolo de administración de red simple [SNMP].

cortafuegos de próxima generación (NGFW): avances en la tecnología de cortafuegos, desde el reconocimiento de aplicaciones, el filtrado basado en el usuario y la prevención de intrusiones hasta la inspección de la nube.

escaneo no acreditado: un análisis que utiliza menos permisos y muchas veces solo puede encontrar parches o actualizaciones faltantes.

acuerdo de confidencialidad (NDA): acuerdo que estipula que las entidades no compartirán información, conocimientos ni materiales confidenciales con terceros no autorizados.

datos no legibles por humanos: información almacenada en un archivo que los seres humanos no pueden leer sin un procesador especializado para decodificar la estructura binaria o compleja.

no repudio: el objetivo de seguridad de garantizar que la parte que envió una transmisión o creó datos permanezca asociada con esos datos y no pueda negar el envío o la creación de esos datos.

proxy no transparente: un servidor que redirige las solicitudes y respuestas de los clientes configurados con el puerto y la dirección de proxy.

autenticación NT LAN Manager (NTLM authentication): un protocolo de autenticación de desafío y respuesta creado por Microsoft para el uso en sus productos.

ofuscación: una técnica que básicamente oculta o camufla el código u otra información para que sea más difícil de leer por usuarios no autorizados.

probabilidad objetiva: la medida matemática de la posibilidad de que ocurra un riesgo.

desvinculación: proceso de asegurar que todos los requisitos de RR. HH. y otros están cubiertos cuando un empleado deja una organización.

prueba de penetración ofensiva: el equipo hostil o atacante en una prueba de penetración o ejercicio de respuesta a incidentes.

copia de seguridad fuera de las instalaciones: una copia de seguridad que escribe los datos del trabajo en un soporte de almacenamiento que se encuentra en una ubicación física separada del sistema de producción.

incorporación: proceso de incorporación de un nuevo empleado, contratista o proveedor.

contraseña de un solo uso (OTP): una contraseña que se genera para usarla en una sesión específica y deja de ser válida después de que finalice la sesión.

protocolo de estado de certificados en línea (OCSP): permite a los clientes solicitar el estado de un certificado digital, para verificar si fue revocado.

ataque en ruta: ataque en el que el actor de amenaza realiza una conexión independiente entre dos víctimas y es capaz de leer y posiblemente modificar el tráfico.

locales: software o servicios instalados y gestionados en la infraestructura informática de un cliente, en lugar de en la nube o estar alojados por un proveedor externo.

red local: una instalación de red privada que pertenece y se opera a través de una organización para uso exclusivo de sus empleados.

copia de seguridad en las instalaciones: una copia de seguridad que escribe los datos del trabajo en un soporte de almacenamiento que se encuentra en la misma ubicación física que el sistema de producción.

Opal: estándares para implementar el cifrado de dispositivos en dispositivos de almacenamiento.

autorización abierta (OAuth): un estándar para la administración de identidades federadas, que permite que los servidores de recursos o los sitios de consumo trabajen con cuentas de usuario creadas y administradas en un proveedor de identidades independiente.

libro mayor público abierto: un registro público distribuido de transacciones que respalda la integridad de las cadenas de bloques.

inteligencia de fuentes abiertas (OSINT): información disponible en forma pública y las herramientas utilizadas para agregarla y buscarla.

orden de volatilidad: el orden en que se deben recuperar los datos volátiles de varias ubicaciones de almacenamiento y dispositivos después de que se produzca un incidente de seguridad.

crimen organizado: un tipo de actor de amenaza que utiliza hacking y fraude informático para obtener beneficios comerciales.

gestión fuera de banda (OOB): acceso a la interfaz administrativa de un dispositivo de red utilizando una red separada de la red de datos habitual. Esto podría utilizar una VLAN separada o un tipo de enlace diferente, como un módem de acceso telefónico.

registro fuera de ciclo: un indicador de posible actividad maliciosa donde las fechas o marcas de tiempo de los eventos no son coherentes.

monitoreo de paquetes: técnicas y herramientas diseñadas para mitigar los riesgos de las vulnerabilidades de las aplicaciones en el código de terceros, como las bibliotecas y las dependencias.

análisis de paquetes: análisis de los datos de carga útil y encabezados de una o más tramas en el tráfico de red capturado.

cortafuegos de filtrado de paquetes: una tecnología de cortafuegos de capa 3 que compara los encabezados de los paquetes con las ACL para determinar qué tráfico de red aceptar.

pruebas de procesamiento paralelo: ejecución simultánea de sistemas primarios y de respaldo para validar la funcionalidad y el rendimiento de los sistemas de copia de seguridad sin interrumpir el funcionamiento normal.

reconocimiento pasivo: técnicas de pruebas de penetración que no interactúan directamente con los sistemas objetivo.

control de seguridad pasivo: un análisis de enumeración, vulnerabilidad o detección de incidentes que analiza solo el tráfico de red interceptado en lugar de enviar sondas a un objetivo. En pocas palabras, las técnicas de reconocimiento pasivo son aquellas que no requieren una interacción directa con el objetivo.

ataque de contraseña: cualquier ataque en el que el atacante intenta obtener acceso y uso no autorizado de las contraseñas.

procedimientos recomendados para contraseñas: reglas para regular la selección y el mantenimiento seguros de los secretos de autenticación del factor de conocimiento, como la longitud, la complejidad, la antigüedad y la reutilización.

administrador de contraseñas: software que puede sugerir y almacenar contraseñas de sitios y aplicaciones para reducir los riesgos derivados de las malas elecciones y el comportamiento de los usuarios. La mayoría de los navegadores tienen un administrador de contraseñas integrado.

rociado de contraseñas: ataque de fuerza bruta en el que se prueban varias cuentas de usuario con un diccionario de contraseñas comunes.

sin contraseña: esquema de autenticación multifactor que utiliza factores de propiedad y biométricos, pero no factores de conocimiento.

parche: una pequeña unidad de código suplementario destinada a abordar un problema de seguridad o una falla de funcionalidad en un paquete de software o sistema operativo.

gestión de parches: identificar, examinar e implementar actualizaciones de aplicaciones y del sistema operativo. Los parches a menudo se clasifican como críticos, críticos para la seguridad, recomendados y opcionales.

Estándar de seguridad de datos de la industria de tarjetas de pago (PCI DSS): estándar de seguridad de la información para organizaciones que procesan pagos con tarjeta bancaria o de crédito.

prueba de penetración: prueba que utiliza herramientas activas y utilidades de seguridad para evaluar la seguridad mediante la simulación de un ataque a un sistema. Una prueba de penetración verificará que existe una amenaza, luego probará activamente y eludirá los controles de seguridad y, por último, explotará las vulnerabilidades del sistema.

codificación porcentual: un mecanismo para codificar caracteres como valores hexadecimales delimitados por el signo de porcentaje.

secreto directo perfecto (PFS): una característica del cifrado de transporte que garantiza que, si una clave se ve comprometida, esto solo afectará a una sola sesión y no facilitará la recuperación de datos de texto sin formato de otras sesiones.

permisos: ajustes de seguridad que controlan el acceso a los objetos, incluidos los elementos del sistema de archivos y los recursos de red.

persistencia (equilibrio de carga): en el equilibrio de carga, la opción de configuración que permite a un cliente mantener una conexión con un servidor de equilibrio de carga durante la sesión. También se conocen como sesiones fijas.

red de área personal [PAN]: alcance de red que usa tecnologías inalámbricas de corto alcance (generalmente basada en Bluetooth o NFC) para establecer las comunicaciones entre dispositivos personales, como teléfonos inteligentes, computadoras portátiles e impresoras/dispositivos periféricos.

número de identificación personal [PIN]: número utilizado junto con los dispositivos de autenticación, como las tarjetas inteligentes; como el PIN solo lo debe conocer el usuario, la pérdida de la tarjeta inteligente no debería representar un riesgo para la seguridad.

pharming: un ataque de suplantación de identidad en el que una solicitud de un sitio web, por lo general un sitio de comercio electrónico, se redirige a un sitio web de aspecto similar, pero falso.

phishing: ataque de ingeniería social basado en el correo electrónico, en el que el atacante envía un correo electrónico desde una fuente supuestamente fiable, como un banco, para intentar obtener información privada de la víctima.

ataque físico: un ataque dirigido contra la infraestructura de cableado, los dispositivos de hardware o el entorno de las instalaciones del sitio que alberga una red.

pruebas de penetración física: técnicas de evaluación que se extienden al sitio y otros sistemas de seguridad física.

pivote: cuando un atacante utiliza un host comprometido (el pivote) como plataforma desde la cual propagar un ataque a otros puntos de la red.

plataforma como un servicio [PaaS]: modelo de servicio en la nube que proporciona servicios de aplicaciones y bases de datos como plataforma para el desarrollo de aplicaciones.

libro de instrucciones: una lista de acciones que se deben realizar para detectar y responder a un tipo específico de incidente.

módulo de autenticación conectable (PAM): un marco para implementar proveedores de autenticación en Linux.

Protocolo de tunelización de punto a punto (PPTP): desarrollado por Cisco y Microsoft para admitir VPN a través de PPP y TCP/IP. El protocolo PPTP es muy vulnerable a ataques de descifrado de contraseñas y se considera obsoleto.

política: un conjunto de reglas estrictamente aplicables que determina cómo debe completarse una tarea.

creación de reflejo de puertos (SPAN): copiado de las comunicaciones de entrada o salida de uno o varios puertos del switch a otro puerto. Esto se utiliza para monitorear las comunicaciones que pasan por el switch.

Protocolo de oficina de correo [POP]: protocolo de aplicación que permite a un cliente descargar mensajes de correo electrónico desde el buzón de un servidor a un cliente a través del puerto TCP/110 o el puerto seguro TCP/995.

programa potencialmente no deseado [PUP]: un software que no se puede clasificar como malicioso de manera definitiva, pero que puede no haber sido elegido o deseado por el usuario.

unidad de distribución de energía (PDU): toma de corriente avanzada que proporciona voltaje de salida filtrado. Una unidad gestionada que admite la gestión remota.

falla de alimentación: pérdida total de la potencia del edificio.

preparación: un proceso de respuesta a incidentes que refuerza los sistemas, define políticas y procedimientos, establece líneas de comunicación y pone recursos en su lugar.

clave previamente compartida (PSK): modo de autenticación de red inalámbrica en el que se utiliza un mecanismo basado en una frase de contraseña para permitir la autenticación de grupo en una red inalámbrica. La frase de contraseña se utiliza para derivar una clave de cifrado.

pretexto: una táctica de ingeniería social en la que un equipo comunicará, directa o indirectamente, una mentira o una verdad a medias para conseguir que alguien crea una falsedad.

control preventivo: tipo de control de seguridad que actúa antes de un incidente para eliminar o reducir la probabilidad de que un ataque tenga éxito.

nube privada: nube que se implementa para el uso de una sola entidad.

clave privada: en el cifrado asimétrico, la clave privada solo la conoce el titular y está vinculada a una clave pública distribuida a aquellos con los que el titular quiere comunicarse de manera segura, pero no se puede derivar de ella. Se puede usar una clave privada para cifrar datos que se pueden descifrar mediante la clave pública vinculada o viceversa.

escalamiento de privilegios: práctica que consiste en explotar los fallos de un sistema operativo u otra aplicación para obtener un nivel de acceso superior al previsto para el usuario o la aplicación.

administración de acceso con privilegios (PAM): políticas, procedimientos y software de respaldo para la gestión de cuentas y credenciales con permisos de administrador.

probabilidad: en el análisis cuantitativo de riesgos, la probabilidad de un evento que se expresa como un porcentaje.

procedimiento: instrucciones detalladas para completar una tarea de manera que cumpla con las políticas y los estándares.

parte interesada en el proyecto: una persona que tiene un interés comercial en el resultado de un proyecto o participa activamente en su trabajo.

información de propiedad exclusiva: información creada por una organización, por lo general, sobre los productos o servicios que fabrica u ofrece.

procedencia: en investigación forense digital, la capacidad de relacionar la fuente de evidencia con la escena de un crimen y demostrar que no fue manipulada.

aprovisionamiento: el proceso de implementación de una cuenta, host o aplicación en un entorno de producción de destino. Esto implica probar la identidad o integridad del recurso y emitirlo con credenciales y permisos de acceso.

lector de proximidad: un escáner que lee datos de una etiqueta RFID o NFC cuando se encuentra dentro del alcance.

servidor proxy: un servidor que media las comunicaciones entre un cliente y otro servidor. Puede filtrar y, a menudo, modificar las comunicaciones, así como proporcionar servicios de almacenamiento en caché para mejorar el rendimiento.

nube pública: nube que se implementa para uso compartido por múltiples inquilinos independientes.

clave pública: durante el cifrado asimétrico, esta clave se distribuye libremente y puede utilizarse para hacer la operación de cifrado o descifrado inverso de la clave privada vinculada en la sincronización.

estándares de criptografía de clave pública (PKCS): una serie de normas que definen el uso de autoridades de certificación y certificados digitales.

infraestructura de clave pública (PKI): marco de autoridades de certificación, certificados digitales, software, servicios y otros componentes criptográficos desplegados con el fin de validar las identidades de los sujetos.

análisis cualitativo de riesgos: el proceso de determinar la probabilidad de ocurrencia y el impacto de los riesgos identificados mediante el uso de razonamiento lógico cuando los datos numéricos no están fácilmente disponibles.

análisis cuantitativo de riesgos: un método numérico que se utiliza para evaluar la probabilidad e impacto de un riesgo y medir su impacto.

cuestionarios: en la gestión de proveedores, medios estructurados para obtener información consistente, lo que permite un análisis y comparación de riesgos más efectivos.

condición de carrera: una vulnerabilidad de software cuando el resultado de los procesos de ejecución depende directamente del orden y el tiempo de ciertos eventos, y dichos eventos no se ejecutan en el orden y tiempo previstos por el desarrollador.

identificación de la frecuencia de radio [RFID]: medios para codificar información en etiquetas pasivas, que pueden activarse y leerse mediante ondas de radio desde un dispositivo lector.

ransomware: malware que intenta extorsionar a la víctima bloqueando el funcionamiento normal de una computadora o encriptando los archivos de la víctima y exigiendo el pago.

tiempo de reacción: el tiempo transcurrido entre la ocurrencia de un incidente y la implementación de una respuesta.

sistema operativo en tiempo real (RTOS): un tipo de sistema operativo que prioriza la ejecución determinista de las operaciones para garantizar una respuesta consistente para las tareas urgentes.

reconocimiento: acciones realizadas para recabar información sobre los sistemas informáticos y el software de una persona u organización. Suele implicar la recopilación de información como los tipos de sistemas y software utilizados, información sobre la cuenta del usuario, tipos de datos y configuración de la red.

recuperación: un proceso de respuesta a incidentes donde los hosts, las redes y los sistemas vuelven a una configuración de referencia segura.

Objetivo de punto de recuperación (RPO): período más largo que una organización puede tolerar que los datos perdidos sean irrecuperables.

objetivo de tiempo de recuperación (RTO): tiempo máximo permitido para restaurar un sistema después de un evento de falla.

redundancia: aprovisionamiento excesivo de recursos a nivel de componente, host o sitio para que haya una conmutación por error a una instancia de trabajo en caso de un problema.

datos regulados: información que tiene requisitos de cumplimiento de almacenamiento y manejo definidos por la legislación nacional y estatal o las regulaciones de la industria.

acceso remoto: infraestructura, protocolos y software que permiten que un host se una a una red local desde una ubicación físicamente remota, o que permiten que una sesión en un host se establezca a través de una red.

troyano de acceso remoto (RAT): malware que crea un canal de administración remota de puerta trasera para permitir que un actor de amenaza acceda y controle el host infectado.

Servicio de autenticación remota de usuario por acceso telefónico [RADIUS]: protocolo AAA que se usa para administrar las infraestructuras de autenticación remota e inalámbrica.

ejecución remota de código (RCE): una vulnerabilidad que permite a un atacante transmitir código desde un host remoto para su ejecución en un host de destino o un módulo que aprovecha dicha vulnerabilidad.

Protocolo de escritorio remoto [RDP]: protocolo de aplicación para operar conexiones remotas a un host usando una interfaz gráfica. El protocolo envía datos de pantalla desde el host remoto al cliente y transfiere la entrada del teclado y el mouse del cliente al host remoto. Usa el puerto TCP 3389.

ataque de repetición: un ataque donde el atacante intercepta algunos datos de autenticación y los reutiliza para intentar restablecer una sesión.

replicación: copia automática de datos entre dos sistemas de procesamiento, ya sea simultáneamente en ambos sistemas (sincrónico) o de una ubicación primaria a una secundaria (asincrónico).

informes: un proceso forense que resume el contenido significativo de datos digitales utilizando métodos y herramientas abiertos, repetibles e imparciales.

transferencia de estado representacional (REST): un estilo arquitectónico estandarizado y sin estado utilizado por las aplicaciones web para la comunicación e integración.

inteligencia sobre amenazas de reputación: listas de bloqueo de fuentes de amenazas conocidas, como firmas de malware, rangos de direcciones IP y dominios DNS.

riesgo residual: riesgo que permanece incluso después de que se implementan los controles.

resiliencia: la capacidad de un sistema o red para recuperarse rápidamente de eventos de falla sin intervención manual o con una intervención manual mínima.

consumo de recursos: un indicador potencial de actividad maliciosa donde el uso de la CPU, la memoria, el almacenamiento o la red se desvía de las normas esperadas.

inaccesibilidad de los recursos: un indicador potencial de actividad maliciosa donde un archivo o recurso de servicio que debería estar disponible es inaccesible.

recursos/financiamiento: la capacidad de los actores de amenazas para recurrir a los fondos para adquirir personal y herramientas y desarrollar nuevos tipos de ataques.

matriz de responsabilidades: identifica que la responsabilidad de la implementación de la seguridad a medida que las aplicaciones, los datos y las cargas de trabajo se transfieren a una plataforma en la nube se comparten entre el cliente y el proveedor de servicios en la nube [CSP].

programa de divulgación responsable: un proceso que permite a los investigadores y revisores revelar de forma segura las vulnerabilidades a un desarrollador de software.

capacidad de respuesta: la capacidad de un sistema para procesar una tarea o carga de trabajo dentro de un período de tiempo aceptable.

proxy reverso: tipo de servidor proxy que protege a los servidores del contacto directo con las peticiones de los clientes.

derecho al olvido: principio de datos de privacidad regulados que protege la capacidad del titular de los datos para solicitar su eliminación.

riesgo: la probabilidad y el impacto (o consecuencia) de un actor de amenaza que provoca una vulnerabilidad.

aceptación del riesgo: respuesta con la que se determina si un riesgo está dentro del ámbito de la organización y no se necesitan más contramedidas que el monitoreo continuo.

análisis de riesgos: proceso para calificar o cuantificar la probabilidad y el impacto de un factor.

apetito de riesgo: una evaluación estratégica de qué nivel de riesgo residual es aceptable para una organización.

evaluación de riesgos: el proceso de identificar riesgos, analizarlos, desarrollar una estrategia de respuesta y mitigar su impacto futuro.

evitación de riesgos: en la mitigación de riesgos, la práctica de cesar una actividad que presenta riesgos.

disuasión de riesgos: en mitigación de riesgos, la respuesta de implementar controles de seguridad para reducir la probabilidad o el impacto de un escenario de amenaza.

excepción de riesgo: categoría de gestión de riesgos que utiliza controles de mitigación alternativos para controlar un factor de riesgo aceptado.

exención de riesgos: categoría de gestión de riesgos que acepta un factor de riesgo no mitigado.

identificación de riesgos: dentro de la evaluación general de riesgos, el proceso específico de enumerar las fuentes de riesgo debido a amenazas y vulnerabilidades.

gestión de riesgos: proceso cíclico de identificación, evaluación, análisis y respuesta a los riesgos.

mitigación de riesgos: respuesta con la que se reduce el riesgo para ajustarse a la disposición de una organización a aceptar riesgos.

responsable del riesgo: un individuo que es responsable de desarrollar e implementar una estrategia de respuesta al riesgo para un riesgo documentado en un registro de riesgos.

registro de riesgos: documento donde se destacan los resultados de las evaluaciones de riesgos en un formato fácil de comprender (como una cuadrícula tipo semáforo). Su propósito es que los gerentes de los departamentos y los técnicos comprendan los riesgos asociados con los flujos de trabajo que administran.

informe de riesgos: un resumen periódico de información relevante sobre los riesgos actuales de un proyecto. Proporciona una descripción resumida de los riesgos conocidos, los riesgos realizados y su impacto en la organización.

umbral de riesgo: límite para los tipos o niveles de riesgo que pueden aceptarse.

tolerancia al riesgo: determina los umbrales que separan los diferentes niveles de riesgo.

transferencia de riesgos: en la mitigación de riesgos, la respuesta con la que se traslada o comparte la responsabilidad del riesgo a otra entidad, por ejemplo mediante la adquisición de un seguro de ciberseguridad.

control de acceso basado en funciones (RBAC): un modelo de control de acceso en el que los recursos están protegidos por las lista de control de acceso (ACL) que son gestionadas por administradores y que proporcionan permisos a los usuarios basados en funciones de trabajo.

análisis de causa raíz: técnica utilizada para determinar la verdadera causa del problema que, una vez eliminada, impide que vuelva a producirse.

autoridad de certificación raíz: en la PKI, una CA que emite certificados a las CA intermedias en una estructura jerárquica.

rooting (rooteo): obtener acceso de nivel de superusuario a través de un dispositivo móvil basado en Android.

firewall de enrutador: un dispositivo de hardware que tiene la función principal de un enrutador, pero que también tiene una funcionalidad de cortafuegos integrada en el firmware del enrutador.

control de acceso basado en reglas: una técnica de control de acceso no discrecional que se basa en un conjunto de reglas operativas o restricciones para hacer cumplir una política de permisos con privilegios mínimos.

reglas de intervención (ROE): una definición de cómo se ejecutará una prueba de penetración y qué restricciones se implementarán. Esto proporciona a quien realiza la prueba de penetración pautas para consultar mientras se llevan a cabo, así no necesitan pedir permiso constantemente a la gerencia para hacer algo.

sal: una contramedida de seguridad que mitiga el impacto de los ataques a tablas hash precalculadas agregando un valor aleatorio (salting) a cada entrada de texto sin formato.

sandbox: entorno informático aislado de un sistema host para garantizar que el entorno se ejecuta de forma controlada y segura. Los enlaces de comunicación entre el sandbox y el host por lo general están absolutamente prohibidos para que el malware o el software defectuoso puedan analizarse de forma aislada y sin riesgo para el host.

desinfección: proceso que consiste en eliminar los datos de un medio de almacenamiento de forma exhaustiva y completa para que no se puedan recuperar los restos de los archivos.

Ley Sarbanes-Oxley (SOX): una ley promulgada en 2002 que dicta los requisitos para el almacenamiento y la retención de documentos relacionados con las operaciones financieras y comerciales de una organización.

escalabilidad: propiedad por la cual un ambiente informático es capaz de satisfacer con éxito las crecientes necesidades de recursos.

subred filtrada: segmento aislado del resto de una red privada por uno o varios cortafuegos que acepta conexiones desde Internet a través de puertos designados.

Perímetro de Servicio de Acceso Seguro (SASE): arquitectura de red y seguridad que proporciona acceso seguro a aplicaciones y servicios en la nube reduciendo la complejidad. Combina servicios de seguridad, como cortafuegos, gestión de identidades y accesos, y una puerta de enlace web segura con servicios de red como SD-WAN.

línea de base segura: guías de configuración, puntos de referencia y mejores prácticas para implementar y mantener un dispositivo de red o un servidor de aplicaciones en un estado seguro para su función determinada.

enclave seguro: extensiones de CPU que protegen los datos almacenados en la memoria del sistema para que un proceso que no es confiable no pueda leerlos.

Protocolo seguro de transferencia de archivos (SFTP): versión segura del Protocolo de Transferencia de Archivos que utiliza un túnel shell seguro (SSH) como método de cifrado para transferir, acceder y gestionar archivos.

algoritmo de hash seguro (SHA): un algoritmo de hash criptográfico creado para tratar las posibles debilidades en MDA. La versión actual es SHA-2.

Shell seguro (SSH): protocolo de aplicación que admite la tunelización segura y emulación de terminal remoto y copia de archivos. SSH funciona por medio del puerto TCP 22.

lenguaje de marcado para aserción de seguridad (SAML): formato de datos basado en XML que se utiliza para intercambiar información de autenticación entre un cliente y un servicio.

Protocolo de automatización de contenidos de seguridad (SCAP): framework del NIST que describe diversas prácticas aceptadas para automatizar el escaneo de vulnerabilidades.

control de seguridad: una tecnología o procedimiento implementado para mitigar vulnerabilidades y riesgos y garantizar la confidencialidad, integridad y disponibilidad (CIA) de la información.

identificador de seguridad (SID): valor asignado a una cuenta por Windows que el sistema operativo utiliza para identificar esa cuenta.

Gestión de información y eventos de seguridad (SIEM): solución que proporciona un análisis en tiempo real o casi real de las alertas de seguridad generadas por el hardware y las aplicaciones de la red.

clave de seguridad: HSM portátil con una interfaz de computadora, como USB o NFC, que se utiliza para la autenticación multifactor.

registro de seguridad: un objetivo para los datos de eventos relacionados con el control de acceso, como la autenticación de usuario y el uso de privilegios.

zona de seguridad: área de la red (o de una red conectada) en la que la configuración de seguridad es la misma para todos los hosts dentro de ella. En seguridad física, un área separada por barreras que controlan los puntos de entrada y salida.

Security-Enhanced Linux (SELinux): el esquema de permisos predeterminado basado en el contexto proporcionado con CentOS y Red Hat Enterprise Linux.

selección de controles efectivos: el proceso de elegir el tipo y la ubicación de los controles de seguridad para garantizar los objetivos de la tríada CIA y el cumplimiento de los requisitos del marco.

unidad de autocifrado (SED): unidad de disco en la que el controlador puede cifrar automáticamente los datos que están escritos en ella.

certificado autofirmado: un certificado digital que fue firmado por la entidad que lo emitió, en lugar de por una CA.

Marco de directivas de remitente [SPF]: registro DNS que identifica hosts autorizados para enviar correos electrónicos para el dominio.

sensor: un monitor que registra (o identifica) datos de los marcos a medida que pasan por los medios de red, mediante métodos como un puerto espejo o un dispositivo TAP.

sensor (alarmas): un componente en un sistema de alarma que identifica la entrada no autorizada mediante la detección de cambios términos o movimiento mediante infrarrojos, ultrasonidos, microondas o presión.

sin servidor: arquitectura de software que ejecuta funciones dentro de contenedores de tiempo de ejecución virtualizados en una nube en lugar de en instancias de servidor exclusivas.

computación sin servidor: características y capacidades de un servidor sin necesidad de realizar tareas de administración de este. La informática sin servidor descarga la gestión de la infraestructura en el proveedor de servicios en la nube, por ejemplo, configurar la capacidad de almacenamiento de archivos sin necesidad de construir y desplegar primero un servidor de archivos.

lado del servidor: en una aplicación web, ingresar datos que se ejecutan o validan como parte de un script o proceso que se ejecuta en el servidor.

falsificación de solicitudes del lado del servidor (SSRF): ataque en el que un atacante se aprovecha de la confianza establecida entre el servidor y los recursos a los que puede acceder, incluido él mismo.

interrupción del servicio: tipo de ataque que compromete la disponibilidad de un activo o proceso comercial.

acuerdo de nivel de servicio (SLA): acuerdo que establece los requisitos y expectativas de servicio entre un consumidor y un proveedor.

identificador de conjunto de servicios [SSID]: cadena de caracteres que identifica una determinada LAN inalámbrica (WLAN).

afinidad de sesión: un enfoque de programación utilizado por los balanceadores de carga para dirigir el tráfico a dispositivos que ya establecieron conexiones con el cliente en cuestión.

TI en la sombra: hardware, software o servicios informáticos utilizados en una red privada sin la autorización del propietario del sistema.

código shell: un bloque ligero de código malicioso que explota una vulnerabilidad de software para obtener acceso inicial a un sistema víctima.

carga lateral: instalación de una aplicación en un dispositivo móvil sin usar una tienda de aplicaciones.

detección basada en firmas: un sistema de monitoreo de red que utiliza un conjunto predefinido de reglas proporcionadas por un proveedor de software o personal de seguridad para identificar eventos que son inaceptables.

Protocolo de transferencia de correo simple [SMTP]: protocolo de aplicación utilizado para enviar correo entre hosts a través de Internet. Los mensajes se envían entre servidores a través del puerto TCP 25 o los envía un cliente de correo a través del puerto seguro TCP/587.

Protocolo de administración de red simple [SNMP]: protocolo de aplicación usado para supervisar y manejar los dispositivos de red. SNMP funciona en los puertos UDP 161 y 162 de manera predeterminada.

Protocolo simple de acceso a objetos (SOAP): protocolo de servicios web basado en XML que se utiliza para intercambiar mensajes.

simulación (prueba): una técnica de prueba que replica las condiciones de un escenario de desastre o incidente de seguridad del mundo real.

Autenticación simultánea de iguales (SAE): mecanismo de autenticación personal para redes Wi-Fi introducido con WPA3 para dirigir las vulnerabilidades del método WPA-PSK.

expectativa de pérdida única (SLE): la cantidad que se perdería en una sola ocurrencia de un factor de riesgo particular.

punto único de falla (SPoF): un componente o sistema que causaría una interrupción completa de un servicio si fallara.

inicio de sesión único (SSO): tecnología de autenticación que permite a un usuario autenticarse una vez y recibir autorizaciones para múltiples servicios.

sumidero: una estrategia de mitigación de ataques DoS que dirige el tráfico que está inundando una dirección IP de destino a una red diferente para su análisis.

sondeo del sitio: documentación sobre una ubicación con el fin de construir una infraestructura inalámbrica ideal. A menudo, contiene las ubicaciones óptimas para la colocación de antenas inalámbricas y puntos de acceso con el fin de proporcionar la cobertura requerida para los clientes e identificar las fuentes de interferencia.

skimming: hacer un duplicado de una tarjeta de acceso sin contacto copiando su token de acceso y programando una nueva tarjeta con los mismos datos.

tarjeta inteligente: dispositivo de seguridad similar a una tarjeta de crédito que puede almacenar información de autenticación, como la clave privada de un usuario, en un criptoprocesador integrado.

SMiShing: una forma de phishing que utiliza mensajes de texto SMS para engañar a una víctima para que revele información.

instantánea (copia de seguridad): se utiliza para crear una copia o instancia de la arquitectura completas de una aplicación, disco o sistema. Se utiliza en los procesos de copia de seguridad para restaurar el sistema o el disco de un dispositivo específico en un momento determinado. Una copia de seguridad de instantáneas también se conoce como copia de seguridad de imágenes.

Snort: un NIDS de código abierto. Se requiere una suscripción (“oinkcode”) para obtener conjuntos de reglas actualizados, que permitan que motor de detección identifique las amenazas más recientes. Los no suscriptores pueden obtener conjuntos de reglas creados por la comunidad.

ingeniería social: actividad en la que el objetivo es utilizar el fraude y el engaño para convencer a los usuarios incautos para que proporcionen datos confidenciales o para que infrinjan las directrices de seguridad.

token de autenticación suave: OTP enviado a un número registrado o cuenta de correo electrónico o generado por una aplicación de autenticación como medio de verificación de dos pasos al autenticar el acceso a la cuenta.

software como servicio (SaaS): modelo de servicio en la nube que proporciona a los usuarios servicios de aplicación totalmente desarrollados.

lista de materiales de software (SBOM): inventario de componentes de código de terceros y de código abierto utilizados en una aplicación o paquete.

análisis de composición de software (SCA): herramientas diseñadas para ayudar a identificar código de código abierto y de terceros durante el desarrollo e implementación de software.

WAN definida por software (SD-WAN): servicios que utilizan mecanismos definidos por software y políticas de enrutamiento para implementar túneles virtuales y redes superpuestas sobre múltiples tipos de red de transporte.

ciclo de vida de desarrollo de software (SDLC): los procesos de planificación, análisis, diseño, implementación y mantenimiento que a menudo rigen el desarrollo de software y sistemas.

redes definidas por software [SDN]: API y dispositivos de hardware/virtuales compatibles que permiten programar dispositivos y sistemas de red.

spyware: software que registra información sobre una PC y sus usuarios, a menudo instalado sin el consentimiento del usuario.

configuraciones estándar: en una arquitectura IaC, la propiedad de que una acción de automatización u orquestación siempre produce el mismo resultado, independientemente del estado anterior del componente.

estándares: resultado esperado o estado de una tarea que se realizó de acuerdo con las políticas y procedimientos establecidos. Los estándares pueden determinarse a nivel interno o medirse con marcos externos.

tabla de estado: información sobre sesiones entre hosts recopilada por un cortafuegos con estado.

inspección con estado: técnica utilizada en cortafuegos para analizar paquetes hasta la capa de aplicación en lugar de filtrarlos solo por información de encabezado, lo que permite que el cortafuegos aplique medidas de seguridad más estrictas.

declaración de trabajo (SOW): un documento que define las expectativas para un acuerdo comercial específico.

análisis de código estático: proceso de revisión del código fuente sin compilar, ya sea en forma manual o mediante herramientas automatizadas.

esteganografía: una técnica para ocultar la presencia de un mensaje, a menudo incrustando información dentro de un archivo u otra entidad.

controlador de excepciones estructuradas (SEH): un mecanismo para abordar condiciones de error inesperadas que pueden surgir durante la ejecución del código. El manejo efectivo de errores reduce las posibilidades de que un programa pueda explotarse.

Inyección de lenguaje de consulta estructurado (Inyección de SQL): ataque que inyecta una consulta de base de datos en los datos de entrada dirigidos a un servidor accediendo del lado cliente de la aplicación.

nombre alternativo de asunto (SAN): un campo en un certificado digital que permite identificar a un host por varios nombres o subdominios de host.

supervisión, control y adquisición de datos [SCADA]: tipo de sistema de control industrial que gestiona dispositivos y equipos a gran escala y en múltiples emplazamientos distribuidos en áreas geográficamente extensas desde una computadora host.

suplicante: en la arquitectura de EAP, el dispositivo que solicita acceso a la red.

cadena de suministro: el proceso integral de suministro, fabricación, distribución y, finalmente, entrega de bienes y servicios a un cliente.

Inundación SYN: un ataque DoS en el que el atacante envía numerosas solicitudes SYN a un servidor de destino, con la esperanza de consumir suficientes recursos para evitar la transferencia de tráfico legítimo.

syslog: protocolo de aplicación y formato de registro de eventos que permite que diferentes dispositivos y aplicaciones de software transmitan registros o registros de eventos a un servidor central. Syslog funciona en el puerto UDP 514 de manera predeterminada.

supervisión del sistema: un software que realiza un seguimiento de la salud de los subsistemas de un equipo a través de métricas informadas por el hardware o sensores del sistema. Esto proporciona un servicio de alertas ante fallas como alta temperatura, intrusión en el chasis, entre otros.

auditoría de sistemas/procesos: un proceso de auditoría con un amplio alcance, que incluye la evaluación de la cadena de suministro, la configuración, el soporte, el monitoreo y los factores de ciberseguridad.

ejercicio de simulación: debate sobre situaciones de emergencia e incidentes de seguridad simulados.

tácticas, técnicas y procedimientos (TTP): análisis de ciberataques históricos y acciones de los adversarios.

deuda técnica: costos acumulados por contar con un sistema o producto ineficaz, en lugar de reemplazarlo por uno mejor.

Protocolo temporal de integridad de clave (TKIP): el mecanismo utilizado en la primera versión de WPA para mejorar la seguridad de los mecanismos de cifrado inalámbrico, en comparación con el estándar WEP defectuoso.

puerto de prueba de acceso (TAP): dispositivo de hardware insertado en un tendido de cables para copiar cuadros para su análisis.

anclaje a red: uso del plan de datos celular de un dispositivo móvil para proporcionar acceso a Internet a una computadora portátil o PC. La PC puede estar conectada al dispositivo móvil mediante USB, Bluetooth o Wi-Fi (un punto de acceso inalámbrico móvil).

CA de terceros: en la PKI, una CA pública que emite certificados para varios dominios y en la que los sistemas operativos y los navegadores confían ampliamente como raíz de confianza.

riesgos de terceros: vulnerabilidades que surgen de las dependencias en las relaciones comerciales con proveedores y clientes.

amenaza: posibilidad de que una entidad ejerza una vulnerabilidad (es decir, que vulnere la seguridad).

actor de amenaza: persona o entidad responsable de un evento que fue identificado como incidente de seguridad o como riesgo.

feed de amenazas: firmas y reglas de coincidencia de patrones suministradas a plataformas de análisis como un feed automatizado.

caza de amenazas: técnica de ciberseguridad destinada a detectar la presencia de amenazas que no se descubrieron con el monitoreo normal de la seguridad.

ticket de otorgamiento de tickets (TGT): en Kerberos, un token emitido a una cuenta autenticada para permitir el acceso a servidores de aplicaciones autorizados.

cronología: en el análisis forense digital, herramienta que muestra la secuencia de eventos del sistema de archivos dentro de una imagen de origen en formato gráfico.

tiempo de verificación a tiempo de uso (TOCTOU): la vulnerabilidad potencial que se produce cuando hay un cambio entre el momento en que una aplicación verifica un recurso y el momento en que la aplicación lo utiliza.

restricciones horarias: políticas o ajustes de configuración que limitan el acceso de un usuario a los recursos.

tokenización: método de desidentificación en el que los datos reales se sustituyen por un token único.

secretos comerciales: propiedad intelectual que otorga a una empresa una ventaja competitiva pero que no ha sido registrada con un derecho de autor, marca registrada o patente.

proxy transparente: un servidor que redirige las solicitudes y respuestas sin que el cliente esté configurado explícitamente para usarlo. También se conoce como un proxy forzado o de interceptación.

Seguridad de la capa de transporte [TLS]: protocolo de seguridad que utiliza certificados para la autenticación y el cifrado para proteger las comunicaciones web y otros protocolos de aplicaciones.

red privada virtual de la seguridad de la capa de transporte (TLS VPN): solución de red privada virtual que utiliza certificados digitales para identificar, alojar y establecer túneles seguros para el tráfico de red.

cifrado de transporte/comunicación: esquema de cifrado aplicado a datos en movimiento, como WPA, IPsec o TLS.

análisis de tendencias: el proceso de detección de patrones dentro de un conjunto de datos a lo largo del tiempo y el uso de esos patrones para realizar predicciones sobre eventos futuros o para comprender mejor eventos pasados.

troyano: programa de software malintencionado oculto dentro de un elemento de software aparentemente inocuo. Por lo general, el troyano se usa para intentar comprometer la seguridad de la computadora objetivo.

módulo de plataforma confiable (TPM): especificación para el almacenamiento seguro basado en hardware de claves de cifrado, contraseñas hash y otra información de identificación de usuarios y plataformas.

tunelización: la práctica de encapsular datos de un protocolo para una transferencia segura a través de otra red como Internet.

lenguaje de programación con seguridad de tipos: un programa que impone una estricta verificación de tipos durante la compilación y garantiza que las variables y los datos se utilicen correctamente. Previene las vulnerabilidades relacionadas con la memoria y los ataques de inyección.

typosquatting (errores tipográficos para engañar): ataque en el que un atacante registra un nombre de dominio con un error ortográfico común de un dominio existente, de modo que un usuario que escribe mal una URL en un navegador accede al sitio web del atacante.

evento de subvoltaje: cuando la energía suministrada por el socket eléctrico de pared es insuficiente para permitir que la computadora funcione de manera correcta. Los eventos de subvoltaje son largas caídas en la salida de energía que a menudo ocasionan los circuitos de distribución de red sobrecargados o defectuosos o una falla en la ruta de suministro desde la estación de energía eléctrica hasta un edificio.

administración de amenazas unificadas [UTM]: dispositivos y agentes de seguridad todo en uno que combinan las funciones de cortafuegos, escáner de malware, detección de intrusiones, escáner de vulnerabilidades, prevención de pérdida de datos, filtrado de contenido, etc.

localizador uniforme de recursos [URL]: esquema de direccionamiento a nivel de la aplicación para TCP/IP, que permite el direccionamiento de recursos legibles por personas. Por ejemplo: protocolo://servidor/archivo, donde “protocolo” es el tipo de recurso (HTTP, FTP), “servidor” es el nombre de la computadora (www.microsoft.com) y “archivo” es el nombre del recurso al que desea acceder.

amenaza interna no intencional: un actor de amenazas que causa una vulnerabilidad o expone un vector de ataque sin intención maliciosa.

fuente de alimentación ininterrumpida (UPS): dispositivo alimentado por batería que suministra energía de CA que un dispositivo electrónico puede utilizar en caso de falla de alimentación.

red peligrosa: configuración que expone una gran superficie de ataque, como a través de puertos de servicio abiertos innecesarios, autenticación débil o nula, uso de credenciales predeterminadas o falta de comunicaciones o cifrado seguros.

análisis de comportamiento de usuarios y entidades (UEBA): sistema que puede proporcionar la identificación automatizada de actividades sospechosas por cuentas de usuario y hosts informáticos.

control de versiones: la práctica de garantizar que los activos que conforman un proyecto se gestionen de cerca cuando llega el momento de realizar cambios.

escalamiento vertical de privilegios: cuando un atacante puede realizar funciones que por lo general se asignan a usuarios con funciones superiores y que a menudo se le niegan explícitamente al atacante.

vigilancia con video: control de seguridad física que utiliza cámaras y dispositivos de grabación para monitorear de forma visual la actividad en un área determinada.

red de área local virtual (VLAN): segmento lógico de red que comprende un dominio de difusión establecido mediante una función de los switches administrados para asignar a cada puerto un ID de VLAN. Aunque los hosts en dos VLAN pueden estar conectados de manera física al mismo switch, el tráfico local está aislado para cada VLAN, por lo que deben usar un enrutador para comunicarse.

Computación de red virtual (VNC): herramienta y protocolo de acceso remoto. VNC es la base de la pantalla compartida de macOS.

Nube privada virtual (VPC): segmento de red privada puesto a disposición de un único consumidor de nube en una nube pública.

red privada virtual (VPN): túnel seguro creado entre dos puntos finales conectados a través de una red de transporte no segura (por lo general, Internet).

virtualización: entorno informático en el que pueden instalarse varios sistemas operativos independientes en una única plataforma de hardware y ejecutarse simultáneamente.

virus: código malicioso insertado en una imagen de archivo ejecutable. El código malicioso se ejecuta cuando se abre el archivo y puede entregar una carga útil, como intentar infectar otros archivos.

vishing: ataque de ingeniería social en el que el actor de amenaza extrae información mientras habla por teléfono o aprovecha los servicios de mensajería de voz basados en IP (VoIP).

visualización: un widget que muestra registros o métricas en un formato visual, como un gráfico o una tabla.

vulnerabilidad: debilidad que se pueda activar accidentalmente o aprovechar intencionalmente para provocar una violación de seguridad.

fuente de vulnerabilidades: una lista sincronizable de datos y scripts utilizados para verificar la existencia de vulnerabilidades. También se conocen como complementos o pruebas de vulnerabilidad de la red (NVT).

escáner de vulnerabilidades: hardware o software configurado con una lista de debilidades y exploits conocidas y que puede escanear su presencia en un sistema operativo anfitrión o en una aplicación concreta.

sitio cálido: lugar de procesamiento alternativo que está inactivo o realiza funciones no críticas en condiciones normales, pero que puede convertirse rápidamente en un lugar de operaciones clave si es necesario.

ataque de agujero de riego: un ataque en el que un atacante se dirige a grupos u organizaciones específicos, descubre qué sitios web frecuentan e inyecta código malicioso en esos sitios.

cortafuegos de aplicaciones web (WAF): un cortafuegos diseñado específicamente para proteger el software que se ejecuta en los servidores web y sus bases de datos internas contra ataques de inyección de código y DoS.

filtro web: una aplicación de software o puerta de enlace que filtra las solicitudes del cliente para varios tipos de contenido de Internet (web, FTP, IM, etc.).

Acceso protegido Wi-Fi (WPA): estándares para autenticar y cifrar el acceso a redes Wi-Fi.

Configuración protegida de Wi-Fi (WPS): una característica de WPA y WPA2 que permite el registro en una red inalámbrica en función de un PIN de ocho dígitos.

dominio comodín: en la PKI, un certificado digital que coincidirá con varios subdominios de un dominio principal.

Privacidad equivalente por cable (WEP): un mecanismo heredado para cifrar los datos que se envían a través de una conexión inalámbrica.

tiempo de recuperación del trabajo (WRT): en la recuperación de desastres, tiempo adicional al RTO de los sistemas individuales para realizar la reintegración y las pruebas de un sistema restaurado o actualizado después de un evento.

multiplicador de la fuerza laboral: una herramienta o automatización que aumenta la productividad de los empleados, lo que les permite realizar más tareas con el mismo estándar por unidad de tiempo.

gusano: tipo de malware que se replica entre procesos en la memoria del sistema y puede propagarse a través de conexiones de red cliente/servidor.

bloqueador de escritura: una herramienta de investigación forense para evitar que el dispositivo de captura o análisis o la estación de trabajo cambie los datos en un disco o medio objetivo.

confianza cero: paradigma de diseño de seguridad en el que cualquier petición (de host a host o de contenedor a contenedor) debe ser autenticada antes de ser permitida.

día cero: vulnerabilidad en el software que no está revisada por el desarrollador o un ataque que explota dicha vulnerabilidad.
