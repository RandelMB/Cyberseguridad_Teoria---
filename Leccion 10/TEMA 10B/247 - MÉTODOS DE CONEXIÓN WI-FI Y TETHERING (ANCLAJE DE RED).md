
Por lo general, los dispositivos móviles utilizan de forma predeterminada una conexión Wi-Fi para los datos, si está disponible. Si el usuario establece una conexión a una red corporativa utilizando una seguridad WPA3 robusta, existe un riesgo bastante bajo de ataques de espionaje o en ruta. Los riesgos asociados al Wi-Fi surgen cuando los usuarios se conectan a puntos de acceso abiertos o a puntos de acceso posiblemente fraudulentos que imitan una red corporativa. Estos permiten que el propietario del punto de acceso lance una gran variedad de ataques, incluida la posibilidad de comprometer sesiones con servidores seguros (mediante un ataque de suplantación de DNS, por ejemplo).

Redes de área personal (PAN)
Las redes de área personal (PAN) habilitan la conectividad entre un dispositivo móvil y periféricos. También se pueden establecer redes ad hoc (o punto a punto) entre dispositivos móviles o entre dispositivos móviles y otros dispositivos informáticos. Por lo general, en términos de seguridad corporativa, estas funciones punto a punto deben desactivarse. Un atacante podría aprovechar un dispositivo mal configurado para obtener acceso a la red corporativa mediante una conexión en modo puente.

Wi-Fi ad hoc y Wi-Fi Direct
Las estaciones inalámbricas pueden establecer conexiones punto a punto entre sí en lugar de utilizar un punto de acceso. También puede denominarse red inalámbrica ad-hoc, lo que significa que la red no está disponible de forma permanente. Sin embargo, no existe un respaldo estándar establecido basado en normativas para las redes ad hoc. La organización MITRE desarrolló un proyecto para permitir que los teléfonos inteligentes Android se configuren en una red ad hoc (mitre.org/research/technology-transfer/open-source-software/smartphone-ad-hoc-networking-span).

Wi-Fi Direct permite establecer conexiones uno a uno entre estaciones, aunque en este caso, uno de los dispositivos funciona en realidad como un punto de acceso mediante software (SoftAP). Wi-Fi Direct depende del protocolo WPS (configuración protegida de Wi-Fi), que presenta muchas vulnerabilidades. Android tiene la capacidad de operar como un punto de acceso Wi-Fi Direct, mientras que iOS utiliza un marco de conectividad multipunto patentado. Sin embargo, puede conectar un dispositivo iOS a otro dispositivo que ejecute un punto de acceso virtual Wi-Fi Direct.

También existen productos de malla inalámbrica de proveedores como Netgear y Google que permiten que todo tipo de dispositivos inalámbricos participen en una red punto a punto. Estos productos puede no ser interoperables, aunque son cada vez más compatibles con el estándar EasyMesh (wi-fi.org/discover-wi-fi/wi-fi-easymesh).

Tethering y puntos de acceso inalámbrico (hotspot)

Un teléfono inteligente puede compartir su conexión a Internet con otro dispositivo, como una PC. Cuando el teléfono inteligente comparte esta conexión con varios otros dispositivos a través de Wi-Fi, se le denomina punto de acceso inalámbrico (hotspot). Cuando la conexión se comparte al contectar el teléfono inteligente a una PC mediante un cable USB o con una sola PC a través de Bluetooth, también se denomina tethering. Sin embargo, el término “tethering de Wi-Fi” se utiliza ampliamente para referirse a un punto de acceso inalámbrico. 

Este tipo de funcionalidad por lo general se desactiva cuando el dispositivo está conectado a una red empresarial, ya que podría usarse para eludir mecanismos de seguridad como la prevención de pérdida de datos o las políticas de filtrado de contenido web.
