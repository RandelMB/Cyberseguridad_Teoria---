Seguridad del sistema operativo
La seguridad del sistema operativo abarca muchas prácticas que tienen como objetivo proteger contra el acceso no autorizado, las filtraciones de datos, las infecciones de malware y otras amenazas de seguridad. Se deben tener en cuenta numerosos requisitos y consideraciones al momento de asegurar un sistema operativo debido a que son productos de software complejos y potentes que operan en el núcleo de todos los sistemas de información. Muchos conceptos de seguridad se aplican a la seguridad del sistema operativo, incluidos controles de acceso, mecanismos de autenticación, configuraciones seguras, seguridad de aplicaciones, codificación segura, gestión de parches, protección de puntos de conexión, capacitación de concientización del usuario y monitoreo.

El endurecimientodescribe el proceso de modificar un sistema operativo o aplicación para que opere de forma segura. La necesidad de endurecimiento debe equilibrarse con los requisitos funcionales y la usabilidad, ya que la protección puede tener un impacto negativo en la forma en que funcionan o interactúan las aplicaciones.

Los estándares de mejores prácticas desempeñan un papel fundamental en el endurecimiento de los dispositivos al proporcionar un conjunto estándar de pautas o listas de verificación para configurar los dispositivos de forma segura. Estos lineamientos, comúnmente desarrollados por expertos en ciberseguridad u organizaciones especializadas, ofrecen un punto de partida para los administradores de sistemas, con el fin de garantizar que los dispositivos estén configurados conforme a los estándares de seguridad de la industria. Muchos de los requisitos pueden aplicarse de manera automática a través de una plantilla de línea base de configuración. El principio fundamental es el de mínima funcionalidad: un sistema debe ejecutar únicamente los protocolos y servicios requeridos por usuarios legítimos, sin excederse en ello. Esto reduce la superficie de ataque potencial.

- Las interfaces proporcionan una conexión a la red. Algunas máquinas pueden tener más de una interfaz. Por ejemplo, puede haber interfaces cableadas e inalámbricas o una interfaz de módem. Algunas máquinas pueden venir con una tarjeta de interfaz de red de gestión. Si alguna de estas interfaces no es necesaria, se la debe desactivar de forma explícita en lugar de dejarla sin utilizar.
- Los servicios ofrecen una librería de funciones para distintos tipos de aplicaciones. Algunos servicios admiten características locales del sistema operativo y de las aplicaciones instaladas. Otros servicios admiten conexiones remotas de clientes a servidores de aplicaciones. Los servicios no utilizados deben desactivarse.
- Los puertos de servicio de aplicaciones permiten al software cliente conectarse a aplicaciones a través de una red. Deberían desactivarse o bloquearse en un cortafuegos si no se requiere acceso remoto. Tenga presente que un servidor podría estar configurado con un puerto no estándar. Por ejemplo, un servidor HTTP puede estar configurado para utilizar 8080 en lugar de 80. Por el contrario, el malware puede intentar enviar datos no estándares a través de un puerto abierto. Un sistema de detección de intrusos debería detectar datos de red que no se ajusten al formato de protocolo esperado.
- El almacenamiento persistente conserva los datos de usuario generados por aplicaciones, además de las credenciales almacenadas en caché. El cifrado de discos es fundamental para la seguridad de los datos. Se pueden utilizar unidades de autocifrado para garantizar que todos los datos en reposo se almacenen de manera segura en todo momento.
También es importante establecer un ciclo de mantenimiento para cada dispositivo y mantenerse actualizado respecto a las nuevas amenazas de seguridad y las respuestas correspondientes para los productos de software específicos que está ejecutando.

Estaciones de trabajo
Las estaciones de trabajo operan en la primera línea de las actividades de una organización y presentan preocupaciones únicas con respecto al endurecimiento de los terminales en comparación con otros dispositivos. Debido a la diversidad de tareas y al gran número de aplicaciones asociadas con el uso de las estaciones de trabajo, estas suelen tener una amplia superficie de ataque. Algunas de las prácticas de endurecimiento que se implementan para minimizar esta superficie de ataque son: la eliminación de software innecesario, la limitación de los privilegios administrativos, la gestión estricta de las instalaciones y actualizaciones de aplicaciones, entre otros cambios. Además, dado que son los empleados quienes operan las estaciones de trabajo, resulta fundamental implementar estrategias de seguridad centradas en el usuario. Esto incluye realizar actividades periódicas de capacitación y concientización para educar a los usuarios sobre amenazas como el phishing, así como promover comportamientos seguros, como el uso de contraseñas robustas, el uso responsable de Internet y el manejo cuidadoso de datos confidenciales, entre otras prácticas.

También es indispensable configurar los ajustes de las estaciones de trabajo para incrementar la seguridad, tales como actualizaciones automáticas, bloqueos de pantalla, cortafuegos, protección de puntos de conexión, detección y prevención de intrusiones, aumento del registros, cifrado, monitoreo, entre otras tantas protecciones. Asimismo, la necesidad de proteger los dispositivos periféricos, como los puertos USB, es exclusiva de las estaciones de trabajo. Por lo general se logra mediante el uso de funciones de software de protección de puntos de conexión y la implementación de políticas estrictas de control de dispositivos. Por último, dadas las diversas funciones y responsabilidades asignadas a las diferentes estaciones de trabajo, la segmentación resulta crucial para restringir las comunicaciones y limitar el potencial de propagación de malware o atacantes a través de la red. 

Configuración de línea de base y ajustes de registro
Es habitual contar con líneas base de configuración separadas para clientes de escritorio, servidores de archivos e impresión, servidores de sistema de nombres de dominio (DNS), servidores de aplicaciones, servidores de servicios de directorio y otros tipos de sistemas. En Windows, los ajustes de configuración se almacenan en el registro. En una red de dominios de Windows, cada equipo unido al dominio recibirá configuraciones de políticas de uno o más objetos de directiva de grupo (GPO). Estas configuraciones de directivas se aplican al registro cada vez que un equipo se inicia. Cuando los hosts se gestionan de forma centralizada y ejecutan únicamente aplicaciones y servicios autorizados, debería haber pocas razones para que los valores del registro relacionados con la seguridad cambien. Los derechos para modificar el registro solo deben otorgarse a las cuentas de usuario y de servicio sobre una base de mínimo privilegio. Se puede configurar un sistema de detección de intrusiones basado en host para alertar sobre eventos de registro sospechosos.

Los informes de desviación de las líneas de base implican examinar la configuración real de los hosts a fin de garantizar que sus ajustes de configuración coincidan con la plantilla de línea de base. En las redes de Windows, la herramienta Microsoft Baseline Security Analyzer (MBSA) se utilizaba ampliamente para validar la configuración de seguridad. Esta y otras herramientas de informes de Microsoft se reemplazaron por el kit de herramientas de cumplimiento de seguridad (docs.microsoft.com/en-us/windows/security/threat-protection/security-compliance-toolkit-10).
