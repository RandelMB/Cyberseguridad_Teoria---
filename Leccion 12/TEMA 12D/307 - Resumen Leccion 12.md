Debe poder explicar el proceso y los procedimientos involucrados en una respuesta a incidentes eficaz y el análisis forense digital, incluir las fuentes de datos y las herramientas de gestión de eventos necesarias para las investigaciones, alertas y monitoreo.

DIRECTRICES PARA REALIZAR LA RESPUESTA A INCIDENTES Y EL MONITOREO

Siga estas directrices para desarrollar o mejorar las políticas y procedimientos de respuesta a incidentes:

Identifique objetivos para implementar una respuesta estructurada a incidentes, siguiendo los procesos de preparación, detección, análisis, contención, erradicación, recuperación y lecciones aprendidas.
Preparese para una respuesta eficaz a incidentes creando un CIRT/CERT/CSIRT con políticas y comunicaciones adecuadas, además de herramientas para el manejo de incidentes.
Desarrolle un sistema de clasificación de incidentes y preparar planes de respuesta a incidentes (IRP) y manuales de estrategias para escenarios de incidentes específicos.
Use simulaciones y ejercicios prácticos para probar los planes de respuesta a incidentes.
Configure el SIEM para agregar fuentes de datos apropiadas y desarrollar reglas de correlación para mostrar alertas, indicadores de estado y análisis de tendencias a través de paneles de control:
Fuentes de datos de archivos de registro de hosts (red, sistema, seguridad, respuesta de análisis de vulnerabilidades).
Fuentes de datos de archivos de registro de aplicaciones (DNS, web, VoIP).
Datos de paquetes de red y detección de intrusos.
Estadísticas de tráfico de red y flujo de protocolos.
Integre los procesos de contención, erradicación y recuperación de respuesta a incidentes con procedimientos de cuarentena, recopilación de evidencia forense, análisis de causa raíz, escaneo de vulnerabilidades basado en puntos de referencia/SCAP y la detección de amenazas.
Identifique estrategias estándar para la contención a través del aislamiento y la segmentación.
Desarrolle o adopte un proceso consistente para que los respondedores de incidentes adquieran y conserven datos forenses y de descubrimiento electrónico:
Considere el orden de volatilidad y la posible pérdida de evidencia si un host se apaga o se desactiva.
Registre la recopilación de evidencia mediante videos e interrogar a testigos para recopilar declaraciones.
Implemente herramientas forenses que puedan capturar y validar evidencia de medios persistentes y no persistentes.
Documente el almacenamiento y manejo de evidencia a través de una cadena de custodia.
Identifique las fuentes de datos disponibles para respaldar las investigaciones:
Registros de eventos y metadatos de cortafuegos, aplicaciones, antivirus de punto final y DLP, eventos de seguridad específicos del sistema operativo, IDS/IPS, dispositivos de red, SNMP y escaneo de vulnerabilidades.
Resultados de las capturas de paquetes y el monitoreo del tráfico de NetFlow.
Considere implementar SIEM para agregar y correlacionar fuentes de datos, impulsar alertas de eventos, y crear paneles de monitoreo e informes automatizados. Utilizar el ajuste de alertas para reducir los falsos positivos, sin aumentar los falsos negativos.
