La memoria del sistema son datos volátiles almacenados en módulos de memoria de acceso aleatorio (RAM). Volátil significa que los datos se pierden cuando se retira la energía. Un volcado de memoria del sistema crea un archivo de imagen que se puede analizar para identificar los procesos que se están ejecutando, el contenido de los sistemas de archivos temporales, los datos del registro, las conexiones de red, las claves criptográficas y otros. También puede ser un medio para acceder a los datos que se cifran cuando se almacenan en un dispositivo de almacenamiento masivo.

Una herramienta de hardware o software especializada puede capturar el contenido de la memoria mientras el host se está ejecutando. Lamentablemente, este tipo de herramientas debe estar preinstalada, ya que requiere un controlador de modo kernel para volcar cualquier dato de interés. Existen varias herramientas comerciales para realizar la adquisición de memoria del sistema en Windows. En Linux, el marco de volatilidad incluye una herramienta para instalar un controlador de kernel.
