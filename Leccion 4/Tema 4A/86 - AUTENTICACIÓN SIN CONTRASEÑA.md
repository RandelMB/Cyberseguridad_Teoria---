Con MFA basada en tokens, la cuenta de usuario por lo general aún está configurada con una contraseña. Esto podría usarse como un mecanismo de respaldo o podría haber un proceso de verificación de dos pasos, en el que el usuario debe ingresar su contraseña y luego enviar una OTP.

Sin contraseña significa que todo el sistema de autenticación ya no procesa factores basados en el conocimiento. Las especificaciones de FIDO2 con WebAuthn proporcionan un marco para la autenticación sin contraseña. Básicamente, funciona de la siguiente manera:

El usuario elige un autenticador de roaming, como una clave de seguridad, o un autenticador de plataforma implementado por el sistema operativo del dispositivo, como Windows Hello o Face ID/Touch ID para macOS e iOS. 
El usuario configura un método seguro o un gesto local para confirmar la presencia y autentica el dispositivo. Este gesto podría ser una huella digital, un reconocimiento facial o un PIN. Esta credencial se valida solo a nivel local mediante el autenticador.
El usuario se registra con una aplicación o servicio web, que se conoce como parte de confianza. Para cada nueva parte de confianza, el autenticador genera un par de claves públicas y privadas. El navegador cliente del usuario obtiene la clave pública del autenticador y la registra para asociarla con una cuenta en la parte de confianza. 
Cuando se le presenta un desafío de autenticación, el usuario realiza el gesto local para desbloquear la clave privada. La clave privada se utiliza para firmar una confirmación de que el gesto local funcionó, y que luego se envía a la parte de confianza.
La parte de confianza utiliza la clave pública para verificar la firma y autenticar la sesión de la cuenta.
Al igual que con FIDO U2F, esto proporciona una seguridad similar a la autenticación de tarjetas inteligentes, pero no requiere que las cuentas tengan certificados digitales y PKI, lo que reduce la carga administrativa. FIDO2 WebAuthn mejora FIDO U2F al agregar una interfaz de programación de aplicaciones [API] que permite que las aplicaciones web funcionen sin un elemento de contraseña para la autenticación. La mayoría de los autenticadores FIDO U2F también deben ser compatibles con FIDO2/WebAuthn.

Para que un sistema sin contraseña sea seguro, el autenticador debe ser de confianza y resistente a ataques de suplantación de identidad o clonación. La ratificación es un mecanismo para que un dispositivo autenticador, como una clave de seguridad FIDO o el TPM en una PC o computadora portátil, demuestre que se trata de una raíz de confianza. Cada clave de seguridad se fabrica con un certificado de ratificación y un ID de modelo. Durante el paso de registro, si la parte de confianza requiere ratificación, el autenticador utiliza esta clave para enviar un informe. La parte de confianza puede verificar el informe de ratificación para verificar que el autenticador sea de una marca y modelo conocidos y que admita las propiedades criptográficas que exija la parte de confianza.

Tenga en cuenta que la clave de ratificación no es única; si fuera única, sería fácil identificar a las personas y sería una grave amenaza para la privacidad. En su lugar, identifica una marca y un modelo en particular.
