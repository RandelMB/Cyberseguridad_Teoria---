

Los usuarios a menudo adoptan prácticas deficientes de gestión de credenciales que son difíciles de controlar, como el uso de la misma contraseña para redes corporativas y sitios web de consumidores. Esto hace que la seguridad de la red empresarial sea vulnerable a las violaciones de datos de estos sitios web. Este riesgo se mitiga mediante un password manager (administrador de contraseñas):

El usuario selecciona una aplicación o servicio de administración de contraseñas. La mayoría de los sistemas operativos y navegadores implementan administradores de contraseñas. Algunos ejemplos son Credential Manager (Administrador de credenciales) de Windows y iCloud Keychain de Apple. Si utiliza un administrador de contraseñas de terceros, el usuario instala un complemento para el navegador que haya elegido.
El usuario protege la bóveda de contraseñas con una contraseña maestra. Es probable que la bóveda se almacene en la nube para que se pueda acceder a las cuentas en varios dispositivos, pero algunos administradores de contraseñas solo ofrecen almacenamiento local.
Cuando el usuario crea o actualiza una cuenta, el administrador de contraseñas genera una contraseña aleatoria. Los parámetros se pueden ajustar para cumplir con la política de longitud y complejidad exigida por el sitio.
Posteriormente, cuando el usuario navega por un sitio, el administrador de contraseñas valida la identidad del sitio mediante su certificado digital y presenta una opción para que el usuario complete la contraseña.

Los principales riesgos de los administradores de contraseñas son la selección de una contraseña maestra débil, el compromiso del almacenamiento o los sistemas en la nube del proveedor y los ataques de suplantación de identidad diseñados para engañar al administrador para que complete una contraseña en un sitio suplantado.
