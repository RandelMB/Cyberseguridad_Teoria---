
Un factor de propiedad significa que el usuario posee algún tipo de dispositivo que solo él puede operar. Esto se conoce como autenticador. El autenticador puede generar o recibir un token que identifica y autentica al usuario. Existen tres tipos principales de generación de tokens:

Autenticación basada en certificados: es cuando el solicitante controla una clave privada que puede generar un token firmado único. El proveedor de identidad puede verificar la firma a través de la clave pública. El principal inconveniente de este enfoque es la carga administrativa de la implementación de PKI para emitir certificados digitales.
Contraseña de un solo uso (OTP): es cuando se genera un token mediante algún tipo de función hash en un valor secreto compartido más una semilla de sincronización, como una marca de tiempo (TOTP) o HMAC (HOTP). El token solo se puede utilizar una vez. Se genera un nuevo token para cada decisión de autenticación. Este enfoque aún utiliza un par de claves y el hashing para la seguridad, pero no requiere PKI.
Segundo factor universal (U2F) de Fast Identity Online (identidad rápida en línea, [FIDO]): utiliza un par de claves públicas/privadas para registrar cada cuenta, y así evita la necesidad de comunicar un secreto compartido, que es una debilidad de HOTP y TOTP. La clave privada se bloquea en el dispositivo U2F y firma el token; la clave pública se registra en el servidor de autenticación y verifica el token. Como no se trata de certificados digitales, la solución no depende de PKI.
Un token de autenticación duro o de hardware se genera dentro de un criptoprocesador seguro. El diseño de autenticación significa que no se realiza una transmisión del token en sí. Se pueden utilizar varios autenticadores basados en dispositivos para implementar tokens duros:

Tarjetas inteligentes: implementará la autenticación basada en certificados. La tarjeta inteligente almacena el certificado digital del usuario, la clave privada asociada con el certificado y un número de identificación personal (PIN) que se utiliza para activar la tarjeta. La tarjeta se debe presentar a un lector. Existen tipos de tarjetas de comunicación de campo cercano [NFC] con contacto físico y sin contacto. 
Contraseña de un solo uso (OTP): hace referencia a un criptoprocesador que puede generar un token. Este tipo de token de hardware no necesita una interfaz para conectarse con una computadora; el usuario solo lee el código que se muestra.
Clave de seguridad: hace referencia a un módulo de seguridad de hardware (HSM) portátil con una interfaz de computadora, como USB o NFC. Están más estrechamente asociados con U2F, pero algunos también pueden admitir la autenticación basada en certificados o HOTP/TOTP. Se debe activar una clave de seguridad para mostrar la presencia. Algunas claves solo tienen un botón de activación, pero la mayoría usan un lector biométrico de huellas digitales para una mayor seguridad. También debe configurarse un PIN como un mecanismo de respaldo.

También existen tarjetas inteligentes y fobs más simples que simplemente transmiten un token estático programado en el dispositivo. Por ejemplo, muchos sistemas de entrada a edificios funcionan sobre la base de códigos estáticos. Estos mecanismos son muy vulnerables a los ataques de clonación y reproducción.
