
Un token de autenticación blando o de software es una contraseña de un solo uso generada por el proveedor de identidad y transmitida al solicitante. La OTP se puede enviar a un número de teléfono registrado como un SMS/mensaje de texto o a una cuenta de correo electrónico. Es más probable que este método use tokens basados en contadores, aunque aún tendrán un período de caducidad.

Los tokens blandos enviados por SMS o correo electrónico en realidad no cuentan como un factor de propiedad. Estos sistemas pueden considerarse como una verificación de dos pasos en lugar de MFA. Los tokens son muy vulnerables a la interceptación.

Se puede generar un token OTP blando más seguro a través de una aplicación de autenticación. Se trata de un software instalado en una computadora o teléfono inteligente. El usuario debe registrar cada proveedor de identidad con la aplicación. Por lo general, se utiliza un código de respuesta rápida (QR) que se puede escanear para comunicar el secreto compartido. Cuando se le solicite que se autentique, el usuario debe desbloquear la aplicación de autenticación con las credenciales de su dispositivo para ver el token OTP. Existen menos riesgo de interceptación que con un SMS o un mensaje de correo electrónico, pero como se ejecuta en un dispositivo de uso compartido, existe la posibilidad de que el malware pueda comprometer la aplicación.
