El sistema de nombres de dominio (DNS) resuelve las solicitudes de host y servicios con nombre a direcciones IP. La resolución de nombres es un método de direccionamiento crítico en internet y en redes privadas. Hay muchos ataques potenciales contra el DNS. En el internet público, los ataques pueden usar técnicas de typosquatting (errores tipográficos para engañar) para hacer que las víctimas confundan los sitios maliciosos con los legítimos. El DNS puede explotarse en un ataque DRDoS. Los actores de amenazas también pueden apuntar directamente a los servicios DNS públicos como un medio para realizar un DoS contra un sitio web o recurso en la nube. Por último, el actor de amenazas podría secuestrar un servidor DNS público e insertar registros falsificados, dirigiendo a las víctimas a sitios web fraudulentos.

En una red privada, es probable que un ataque de DNS signifique algún tipo de envenenamiento de DNS. El envenenamiento de DNS vulnera el proceso mediante el cual los clientes consultan los servidores de nombres para localizar la dirección IP de un nombre de dominio. Hay varias maneras en las que se puede perpetrar un ataque de envenenamiento de DNS.

Ataques en ruta basados en DNS
Si el actor de amenazas tiene acceso a la misma red local que la víctima, el atacante puede usar el envenenamiento de ARP para responder a las consultas de DNS de la víctima con respuestas falsificadas. Esto podría combinarse con un ataque de denegación de servicio en el servidor DNS legítimo de la víctima. Se podría usar un DHCP malicioso para configurar clientes con la dirección de un solucionador de DNS controlado por el actor de amenazas.

Envenenamiento de caché de cliente DNS
Antes de que se desarrollara el DNS en la década de 1980, la resolución de nombres se realizaba mediante un archivo de texto llamado HOSTS. Cada asociación de nombre: dirección IP se registraba en este archivo, y los administradores de sistemas tenían que descargar la última copia e instalarla en cada cliente o servidor de internet de forma manual.  Aunque la mayoría de la resolución de nombres ahora funciona a través de DNS, el archivo HOSTS todavía está presente y la mayoría de los sistemas operativos verifican el archivo antes de usar DNS. Su contenido se carga en una caché de asociaciones de nombre:IP conocidas, y el cliente solo se comunica con un servidor DNS si el nombre no está en caché.  Por lo tanto, si un atacante es capaz de colocar una asociación falsa de nombre: dirección IP en el archivo HOSTS y envenenar efectivamente la memoria caché del DNS, podrá redirigir el tráfico. El archivo HOSTS requiere acceso de administrador para modificarlo. En los sistemas Unix y Linux se almacena como /etc/hosts, mientras que en Windows se coloca en %SystemRoot%\System32\Drivers\etc\hosts.  La presencia de entradas sospechosas en el archivo HOSTS es un indicador de que la máquina se ha visto vulnerada.

Envenenamiento de caché del servidor DNS
El envenenamiento de la caché del servidor DNS tiene como objetivo corromper los registros que tiene el propio servidor DNS. Esto se puede lograr realizando un ataque de DoS contra el servidor que contiene los registros autorizados para el dominio y, a continuación, falsificando las respuestas a las solicitudes de otros servidores de nombres. Otro ataque consiste en hacer que el servidor de nombres de la víctima responda a una consulta recursiva del host atacante. Una consulta recursiva obliga al servidor DNS a consultar al servidor autorizado para obtener la respuesta en nombre del cliente. El DNS del atacante, que se hace pasar por el servidor de nombres autorizado, responde con la respuesta a la consulta, pero también incluye muchas asociaciones falsas de dominio: IP para otros dominios que el DNS de la víctima acepta como genuino. La herramienta nslookup o dig puede utilizarse para consultar los registros de nombres y los registros en caché que tiene un servidor para descubrir si se han insertado registros falsos.

Indicadores de ataques a DNS
Un servidor DNS puede registrar un evento cada vez que procesa una solicitud de conversión entre un nombre de dominio y una dirección IP. Los registros de eventos de DNS pueden contener una variedad de información que puede proporcionar inteligencia de seguridad útil e indicadores de ataque, como los siguientes:

Los tipos de consultas que un host ha realizado al DNS.
Los hosts que se comunican con rangos o dominios de direcciones IP sospechosas.
Anomalías estadísticas, como picos o un gran número de fallas en la búsqueda de DNS, que pueden apuntar a computadoras infectadas con malware, mal configuradas o que ejecutan aplicaciones obsoletas o defectuosas.
El DNS también es una opción popular para implementar el comando y control (C&C) de troyanos de acceso remoto. Se puede utilizar como un medio de exfiltración encubierta de datos de una red privada.
