Cuando un usuario elige una contraseña, el valor en texto sin cifrado se convierte en un hash criptográfico. Esto significa que, en teoría, nadie excepto el usuario (ni siquiera el administrador del sistema) conoce la contraseña, porque el texto no cifrado no se debería poder recuperar del hash. Un ataque de contraseña tiene como objetivo explotar las debilidades inherentes a la selección y administración de contraseñas para recuperar el texto no cifrado y usarlo para poner en riesgo una cuenta.

Ataques en línea
Un ataque de contraseña en línea es aquel en el que el atacante interactúa directamente con el servicio de autenticación; un formulario de inicio de sesión web o una puerta de enlace VPN, por ejemplo. Un ataque de contraseña en línea puede aparecer en los registros de auditoría como inicios de sesión fallidos repetidos y luego un inicio de sesión exitoso, o como intentos de inicio de sesión exitosos en momentos o ubicaciones inusuales. Además de asegurarse de que los usuarios usen contraseñas seguras, los ataques de contraseñas en línea pueden mitigarse restringiendo el número o la tasa de intentos de inicio de sesión y evitando los intentos de inicio de sesión desde direcciones IP que se sabe que son incorrectas.

Tenga en cuenta que restringir los inicios de sesión puede convertirse en una vulnerabilidad, ya que expone la cuenta a ataques de denegación de servicio. El atacante intenta autenticarse de manera indefinida, lo que bloquea a los usuarios válidos.

Ataques sin conexión 
Un ataque sin conexión significa que el atacante logró obtener una base de datos de hashes de contraseña, como %SystemRoot%\System32\config\SAM, %SystemRoot%\NTDS\NTDS.DIT (el almacén de credenciales del Active Directory) o /etc/shadow.  Una vez que se obtiene la base de datos de contraseñas, el descifrador de contraseñas no interactúa con el sistema de autenticación. El único indicador de este tipo de ataque (que no sea el uso indebido de la cuenta en caso de un ataque exitoso) es un registro de auditoría del sistema de archivos que registra la cuenta maliciosa que accede a uno de estos archivos. Los actores de amenazas también pueden leer las credenciales de la memoria del host, en cuyo caso el único indicador confiable podría ser la presencia de herramientas de ataque en un host.

Si el atacante no puede obtener una base de datos de contraseñas, se puede usar un rastreador de paquetes para obtener la respuesta del cliente a un desafío del servidor en un protocolo de autenticación. Algunos protocolos envían el hash directamente; otros lo utilizan para derivar una clave de cifrado. Las debilidades en los protocolos que utilizan claves derivadas pueden permitir la extracción del hash para descifrar la contraseña.

Ataques de fuerza bruta
Un ataque de fuerza bruta intenta todas las combinaciones posibles en el espacio de salida para tratar de hacer coincidir un hash capturado y derivar el texto no cifrado que lo generó. El espacio de salida está determinado por el número de bits utilizados por el algoritmo (por ejemplo, MD5 de 128 bits o SHA256 de 256 bits). Cuanto mayor sea el espacio de salida y más caracteres se hayan usado en la contraseña de texto no cifrado, más difícil será calcular y probar cada hash posible para encontrar una coincidencia. Los ataques con fuerza bruta están muy limitados por el tiempo y los recursos informáticos y, por lo tanto, son más eficaces para descifrar contraseñas cortas.  Sin embargo, los ataques de fuerza bruta distribuidos en múltiples componentes de hardware, como un clúster de tarjetas gráficas de alta gama, pueden tener éxito al descifrar contraseñas más largas.

Ataques de diccionario e híbridos
Un ataque de diccionario se puede usar donde existe una buena posibilidad de adivinar el valor probable del texto no codificado, como una contraseña sencilla. El software genera valores hash a partir de un diccionario de textos no cifrados para tratar de hacer coincidir uno con un hash capturado. Un ataque híbrido de contraseña utiliza una combinación de ataques de diccionario y fuerza bruta . Está dirigido principalmente contra contraseñas ingenuas con una complejidad inadecuada, como james1.  El algoritmo de descifrado de contraseñas prueba palabras y nombres de diccionario en combinación con una máscara que limita el número de variaciones a probar, como agregar prefijos y sufijos numéricos. 

Pulverización de contraseñas
La pulverización de contraseñas es un ataque horizontal en línea de fuerza bruta. Significa que el atacante elige una o más contraseñas comunes (por ejemplo, “password” o “123456”) y las prueba junto con varios nombres de usuario.
