El  Simple Network Management Protocol (SNMP) (protocolo de administración de red simple) es un marco ampliamente utilizado para la gestión y el monitoreo. El SNMP consta de un monitor SNMP y agentes.

- El agente es un proceso (software o firmware) que se ejecuta en un switch, enrutador, servidor u otro dispositivo de red compatible con SNMP.
- Este agente mantiene una base de datos llamada base de información de administración (MIB) que contiene estadísticas relacionadas con la actividad del dispositivo (por ejemplo, el número de fotogramas por segundo gestionadas por un switch). El agente también tiene la capacidad de iniciar una operación de captura en la que informa al sistema de administración sobre un evento destacado (por ejemplo, una falla del puerto). El umbral para activar capturas se puede establecer para cada valor. Las consultas del dispositivo se llevan a cabo a través del puerto 161 (UDP); las capturas se comunican a través del puerto 162 (también UDP).
- El monitor SNMP (un programa de software) proporciona una ubicación desde la cual se puede supervisar la actividad de la red. Este supervisa a todos los agentes al sondearlos a intervalos regulares para obtener información de sus MIB y muestra la información para revisión. También muestra cualquier operación de captura como alertas para el administrador de la red para evaluar y actuar según sea necesario.
Si no se utiliza el SNMP, se debe desactivar. A continuación, se incluyen algunas pautas importantes a tener en cuenta al ejecutar el SNMP:

- Los nombres de la comunidad SNMP se envían en texto plano y, por lo tanto, no deben transmitirse a través de la red si existe algún riesgo de que puedan ser interceptados.
- Utilice nombres de comunidad difíciles de adivinar; nunca deje el nombre de la comunidad en blanco ni establezca el valor predeterminado.
- Utilice listas de control de acceso para restringir las operaciones de administración a hosts conocidos (es decir, restringir a una o dos direcciones IP de host).
- Use el SNMP v3 siempre que sea posible y deshabilite las versiones anteriores del SNMP. El SNMP v3 admite el cifrado y la autenticación sólida basada en el usuario. En lugar de nombres de comunidad, los agentes están configurados con una lista de nombres de usuario y permisos de acceso. Cuando se requiere autenticación, los mensajes SNMP se firman con un hash de la frase de contraseña del usuario. El agente puede verificar la firma y autenticar al usuario mediante su propio registro de la frase de contraseña.
