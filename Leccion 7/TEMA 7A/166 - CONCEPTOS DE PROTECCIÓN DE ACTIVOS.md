

Desde la perspectiva de las operaciones de ciberseguridad, los activos describen los recursos críticos, la información y los componentes de infraestructura que deben protegerse de posibles amenazas y accesos no autorizados. El término “activo” abarca recursos como los dispositivos de hardware, las aplicaciones de software, los repositorios de datos, los componentes de red y otros. La protección de activos es fundamental para mantener la integridad, la confidencialidad y la disponibilidad del sistema de información de una organización. 

Los equipos de ciberseguridad son responsables de identificar y priorizar estos activos en función de su sensibilidad y del impacto potencial en las funciones principales de la organización en caso de pérdida, robo o violación. La protección activa de los activos minimiza los posibles impactos de las violaciones de seguridad y reduce la probabilidad de pérdida o daño.

Identificación de activos y convenciones sobre la nomenclatura estándar
Los activos tangibles se pueden identificar utilizando una etiqueta de código de barras o una etiqueta de identificación por radiofrecuencia (RFID) adjunta al dispositivo (o tan solo mediante un número de identificación). Una etiqueta RFID es un chip programado con los datos de los activos. Cuando está dentro del alcance de un escáner, el chip se activa y le envía señales. El escáner alerta al software de administración para que se actualice la ubicación del dispositivo. Además del seguimiento de activos, permite que el software de gestión rastree la ubicación del dispositivo, lo que dificulta el robo.

Una convención de nomenclatura estándar hace que el entorno sea más consistente para los activos de hardware y para los activos digitales, como las cuentas y las máquinas virtuales. La estrategia de nomenclatura debe permitir a los administradores identificar el tipo y la función de cualquier recurso o ubicación en particular en cualquier punto de la base de datos de gestión de configuración (CMDB) o del directorio de red. Cada etiqueta debe cumplir con las reglas para los nombres de host y DNS (support.microsoft.com/en-us/help/909264/naming-conventions-in-active-directory-for-computers-domains-sites-and). Además de un atributo ID, la ubicación y la función de los activos tangibles y digitales se pueden registrar utilizando etiquetas y campos de atributos o registros de recursos DNS CNAME y TXT.

La gestión de la configuración garantiza que cada elemento configurable dentro de un inventario de activos no se haya desviado de su configuración aprobada. El control de cambios y la gestión del cambios reducen el riesgo de que los cambios en estos componentes puedan causar una interrupción en las operaciones de la organización.

ITIL® es un marco de trabajo de documentación popular para actividades y procesos de buenas y mejores prácticas, que se aplica para la prestación de servicios de TI. Bajo ITIL, la gestión de la configuración se implementa mediante los siguientes elementos:

- Los activos de servicio son cosas, procesos o personas que contribuyen a la prestación de un servicio de TI.
- Un elemento de configuración (CI) es un activo que para poder usarlo para prestar un servicio, se deben aplicar procedimientos de gestión específicos. Cada CI debe estar etiquetado, idealmente con una convención de nomenclatura estándar. Los CI se definen por sus atributos y relaciones almacenados en una base de datos de gestión de configuración (CMDB).
- Una configuración de línea de base es una lista de configuraciones que un activo, como un servidor o una aplicación, debe cumplir. Las líneas de base de seguridad describen el conjunto mínimo de ajustes de configuración de seguridad que un dispositivo o software debe mantener para considerarse adecuadamente protegido.
- Un sistema de gestión de configuración (CMS) describe las herramientas y las bases de datos utilizadas para recopilar, almacenar, administrar, actualizar e informar datos sobre los CI. Una red pequeña puede capturar esta información en hojas de cálculo y diagramas, mientras que una organización grande puede invertir en aplicaciones específicas diseñadas para entornos empresariales. 

Los diagramas son la mejor manera de capturar las complejas relaciones entre los elementos de la red. Los diagramas ilustran el uso de CI en flujos de trabajo empresariales, topologías de red lógicas (IP) y físicas, y diseños de racks de red.
