

Las copias de seguridad desempeñan un papel fundamental en la protección de activos ya que garantiza la disponibilidad e integridad de los datos y sistemas críticos de una organización. Dado que consisten en copiar información importante y almacenarla de forma segura en lugares separados, las copias de seguridad ofrecen una red para contingencia en caso de que falle el hardware, se corrompan los datos o se sufran ataques cibernéticos como el ransomware. Es fundamental probar y verificar con regularidad los datos de backup (copia de seguridad) a fin de garantizar la confiabilidad del proceso de recuperación. 

En un entorno empresarial, las técnicas simples de backup (copia de seguridad) a menudo resultan insuficientes para abordar los desafíos y requisitos exclusivos de las grandes organizaciones. La escalabilidad se convierte en una preocupación crítica cuando es necesario gestionar de manera eficiente grandes cantidades de datos. Los métodos de copia de seguridad simples pueden tener dificultades para adaptarse al crecimiento en el tamaño y la complejidad de los datos. 

Los problemas de rendimiento causados por técnicas de copia de seguridad simples pueden interrumpir las operaciones comerciales ya que ralentizan las aplicaciones mientras se ejecutan y, por lo general, tienen tiempos de recuperación prolongados. Además, las empresas exigen una mayor granularidad y personalización para dirigirse a aplicaciones, bases de datos o subconjuntos de datos específicos, que las técnicas simples a menudo no proporcionan. 

Para satisfacer los requisitos de cumplimiento de las normas y seguridad se exigen funciones avanzadas como el cifrado de datos, el control de acceso y las pistas de auditoría de los que suelen carecer los enfoques simplistas. Además, los planes sólidos de recuperación ante desastres y la gestión centralizada son componentes esenciales de una estrategia empresarial de copias de seguridad. Es posible que las técnicas de copia de seguridad simples no admitan funciones avanzadas como la replicación fuera del sitio, la conmutación por error automatizada o la administración optimizada de los diversos sistemas y ubicaciones geográficas que componen el entorno de tecnología de la información de una organización moderna. 

Las funcionalidades críticas para las soluciones de backup (copia de seguridad) empresariales suelen incluir las siguientes características:

- Soporte para varios entornos (virtual, físico y en la nube)
- Desduplicación y compresión de datos para optimizar el espacio de almacenamiento
- Recuperación y replicación instantáneas para una rápida tolerancia a fallos
- Protección contra ransomware y cifrado para la seguridad de los datos
- Opciones de restauración granular para archivos, carpetas o aplicaciones individuales
- Herramientas de informes, monitoreo y alertas para una gestión efectiva
- Integración con plataformas de virtualización, proveedores de nube y sistemas de almacenamiento populares
 La desduplicación de datos describe una técnica de compresión de datos que optimiza el espacio de almacenamiento al identificar y eliminar datos redundantes. Funciona analizando bloques de datos dentro de un conjunto de datos y comparándolos para encontrar bloques idénticos. En lugar de almacenar varias copias de los mismos datos, la desduplicación almacena una sola copia y crea referencias o punteros a esa copia para todas las demás instancias. La desduplicación se puede realizar en diferentes niveles, como a nivel de archivo, a nivel de bloque o a nivel de byte. La desduplicación minimiza significativamente los requisitos de almacenamiento y mejora la eficiencia de la transferencia de datos, en particular en los procesos de copia de seguridad y replicación de datos, al reducir la cantidad de datos duplicados almacenados.

Frecuencia de las copias de seguridad
Muchas dinámicas influyen en los requisitos de la frecuencia de las copias de seguridad de datos, incluidos los requisitos reglamentarios, la volatilidad de los datos, el rendimiento del sistema, las capacidades de la arquitectura y las necesidades operativas. Las organizaciones con datos muy dinámicos o mandatos regulatorios estrictos pueden optar por hacer copias de seguridad más frecuentes para minimizar el riesgo de pérdida de datos y garantizar el cumplimiento. Por el contrario, las empresas con datos relativamente estables o una supervisión regulatoria menos estricta pueden optar por copias de seguridad menos frecuentes, equilibrar la protección de datos, los costos de copia de seguridad de datos y los gastos generales de mantenimiento. Básicamente, la frecuencia óptima de las copias de seguridad se determina mediante la evaluación meticulosa de los requisitos reglamentarios, las necesidades específicas, la tolerancia al riesgo y los recursos de una organización.

Copias de seguridad en las instalaciones o fuera
La necesidad de contar con copias de seguridad en las instalaciones y las copias de seguridad fuera de las instalaciones debe equilibrarse, ya que ambas opciones son fundamentales para asegurar los datos críticos y garantizar la continuidad del negocio. Las copias de seguridad en las instalaciones implican almacenar datos localmente (en el mismo lugar que los sistemas protegidos) en dispositivos como los discos duros o las cintas para proporcionar un acceso y recuperación rápidos en caso de pérdida de datos, corrupción o fallas del sistema. Por otra parte, las copias de seguridad fuera de las instalaciones implican la transferencia de datos a una ubicación remota, para garantizar la protección contra desastres naturales, robos y otras amenazas físicas a la infraestructura local, así como la pérdida catastrófica del sistema que puede resultar de la infección por ransomware, por ejemplo. 

	El ransomware representa una amenaza significativa para las empresas y organizaciones al cifrar datos vitales y exigir un rescate por su liberación. En muchos casos, los ataques de ransomware también se dirigen a la infraestructura de copia de seguridad, lo que dificulta los esfuerzos de recuperación y exacerba aún más el impacto del ataque. Los perpetradores a menudo emplean técnicas avanzadas para infiltrarse y comprometer los sistemas primarios y de respaldo, haciéndolos inútiles cuando son necesarios. Las organizaciones pueden implementar varias estrategias para defenderse de este riesgo, como guardar backups (copias de seguridad) aislados, desconectados físicamente de la red, lo que evita activamente que el ransomware acceda y los encripte.

Validación de recuperación
Las técnicas de validación de recuperación crítica desempeñan un papel vital para garantizar la efectividad y confiabilidad de las estrategias de copia de seguridad. Las organizaciones pueden identificar posibles problemas y debilidades en sus procesos de recuperación de datos probando las copias de seguridad y realizando las mejoras necesarias. Una técnica común es la prueba de recuperación completa, que consiste en restaurar en un entorno separado todo un sistema a partir de una copia de seguridad y verificar que el sistema recuperado sea completamente funcional. Este método ayuda a garantizar que todos los componentes críticos, como los sistemas operativos, las aplicaciones y los datos, se puedan restaurar y funcionen como se espera. 

Otro enfoque es el de la prueba de recuperación parcial. En este caso, los archivos, carpetas o bases de datos seleccionados se restauran para validar la integridad y consistencia de subconjuntos de datos específicos. Las organizaciones pueden realizar auditorías periódicas de copias de seguridad, y con ellas verificar sus registros, horarios y configuraciones a fin de garantizar que las copias de seguridad se creen y mantengan según lo previsto y requerido. Además, la simulación de escenarios de recuperación ante desastres, como las fallas de hardware o los ataques de ransomware, proporciona información valiosa sobre la preparación y la resiliencia de una organización. Las estrategias de validación de recuperación son fundamentales porque las copias de seguridad pueden completarse con “100 % de éxito”, pero enmascaran los problemas hasta que el conjunto de copias de seguridad se utiliza para una recuperación. 

Además, muchas organizaciones desconocen el tiempo necesario para realizar una recuperación, ya que los tiempos de recuperación a menudo son mucho más largos de lo que se supone.
