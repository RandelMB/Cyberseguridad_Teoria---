Deberá ser capaz de resumir los tipos de evaluaciones de seguridad, como la vulnerabilidad, la búsqueda de amenazas y las pruebas de penetración. También deberá poder explicar los procedimientos generales para realizar estas evaluaciones.

DIRECTRICES PARA LA REALIZACIÓN DE EVALUACIONES DE VULNERABILIDAD
Siga estas directrices cuando considere el uso de evaluaciones de seguridad:

- Identifique los procedimientos y las herramientas que se requieren para detectar vulnerabilidades. Podría significar el aprovisionamiento de escáneres de red pasivos, escáneres de red activos remotos o basados en agentes y escáneres de aplicaciones o aplicaciones web.
- Desarrolle un plan de configuración y mantenimiento para garantizar las actualizaciones a las fuentes de vulnerabilidades y amenazas.
- Ejecute escaneos periódicos y revisar los resultados para identificar false positives (falsos positivos) y false negatives (falsos negativos), utilizando la revisión de registros y la información adicional de CVE para validar los resultados en caso de ser necesario.
- Programe revisiones de configuración y planes de corrección, utilizando la criticidad de vulnerabilidad CVSS para priorizar las acciones.
- Considere la implementación de plataformas de análisis de amenazas, así como la supervisión de avisos y boletines para fuentes de amenazas nuevas.
- Considere la implementación de ejercicios de pruebas de penetración, asegurándose de que estos se configuren con un alcance de proyecto claro.
