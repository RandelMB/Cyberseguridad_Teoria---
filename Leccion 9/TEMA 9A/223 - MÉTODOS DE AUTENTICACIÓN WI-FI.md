
Para asegurar una red, se debe confirmar que solo se conectan a ella usuarios válidos. La autenticación Wi-Fi se presenta en tres tipos: personal, abierta y empresarial. Dentro de la categoría personal, existen dos métodos: la autenticación con clave previamente compartida (PSK) y la autenticación simultánea de iguales (SAE).

Autenticación con clave previamente compartida WPA2
En WPA2, la autenticación con clave previamente compartida (PSK) usa una frase de contraseña para generar la clave que se utiliza para cifrar las comunicaciones. También se denomina autenticación de grupo porque un grupo de usuarios comparte el mismo secreto. Cuando el punto de acceso está configurado en modo WPA2-PSK, el administrador configura una frase de contraseña de entre 8 y 63 caracteres ASCII. Esto se convierte en un HMAC de 256 bits (expresado como un valor hexadecimal de 64 caracteres) mediante el algoritmo de estiramiento de claves PBKDF2. Este HMAC se conoce como clave maestra por pares (PMK). Se debe configurar el mismo secreto en el punto de acceso y en cada nodo que se una a la red. La PMK se usa como parte del protocolo de enlace de 4 vías de WPA2 para derivar varias claves de sesión.

Se ha demostrado que todos los tipos de autenticación personal de Wi-Fi son vulnerables a los ataques que permiten ataques de diccionario o de fuerza bruta contra la frase de contraseña. Como mínimo, la frase de contraseña debe tener al menos 14 caracteres para intentar mitigar los riesgos de cracking.

Autenticación personal WPA3
Aunque WPA3 sigue utilizando una frase de contraseña para autenticar las estaciones en modo personal, cambia el método por el que se utiliza este secreto para acordar las claves de sesión. El esquema utilizado se denomina Intercambio de claves autenticado por contraseña [PAKE]. En WPA3, el protocolo autenticación simultánea de iguales (SAE) sustituye al protocolo de enlace de 4 vías, que se descubrió que es vulnerable a varios ataques. SAE utiliza el protocolo de enlace Dragonfly, que consiste básicamente en un acuerdo de claves Diffie-Hellman sobre curvas elípticas, combinado con un valor hash derivado de la contraseña y la dirección MAC del dispositivo para autenticar los nodos. Con SAE, no debería haber forma de que un atacante detecte el protocolo de enlace para obtener el valor hash e intente usar un ataque de fuerza bruta o de diccionario sin conexión para recuperar la contraseña. Dragonfly también implementa claves de sesión efímeras que proporcionan secreto de reenvío.

Las interfaces de configuración para los puntos de acceso pueden usar diferentes etiquetas para estos métodos. Es posible que vea WPA2-Personal y WPA3-SAE en lugar de WPA2-PSK y WPA3-Personal, por ejemplo. Además, un punto de acceso se puede configurar solo para WPA3 o con compatibilidad para WPA2 heredado (modo de transición WPA3-Personal). Los investigadores ya encontraron fallas en WPA3-Personal, una de las que se basa en un ataque de degradación para usar WPA2 (wi-fi.org/security-update-april-2019).

Autenticación avanzada
Los modos de autenticación empresarial, como WPA2/WPA3-Enterprise, incluyen varios componentes esenciales diseñados para mejorar la seguridad de las redes inalámbricas corporativas. Un elemento importante es la autenticación 802.1x, que proporciona un marco de control de acceso a la red basado en puertos, lo que garantiza que solo los dispositivos autenticados tengan acceso a la red. Por lo general, 802.1x requiere un servidor de autenticación como RADIUS (Remote Authentication Dial-In User Service), que verifica las credenciales de los usuarios o dispositivos que intentan conectarse a la red.

En los esquemas de autenticación en modo empresarial, los usuarios tienen un conjunto único de credenciales en lugar de una frase de contraseña compartida como se usa en el modo personal WPA2/WPA3. Exigir que cada usuario o dispositivo se autentique por medio de credenciales únicas permite a los administradores de red realizar un seguimiento del uso de la red a nivel granular. El protocolo también admite múltiples tipos de protocolo de autenticación extensible (EAP), como EAP-TLS, EAP-TTLS o PEAP, que definen métodos de autenticación específicos. Por ejemplo, EAP-TLS utiliza certificados cliente-servidor para la autenticación mutua, mientras que EAP-TTLS y PEAP utilizan un certificado del lado del servidor. El certificado del lado del servidor se utiliza para establecer un túnel seguro para transmitir las credenciales del usuario y ayuda a los dispositivos a validar la legitimidad del punto de acceso. La autenticación en modo empresarial incluye la gestión dinámica de claves de cifrado, que cambia de forma automática las claves de cifrado que se utilizan durante la sesión de un usuario. 

Servicio de autenticación remota de usuario por acceso telefónico (RADIUS)
El Servicio de autenticación remota de usuario por acceso telefónico [RADIUS] se publica como un estándar de Internet. Existen varios productos del servidor RADIUS y cliente.

El dispositivo NAS (cliente RADIUS) se configura con la dirección IP del servidor de RADIUS y con un secreto compartido. Esto permite que el cliente se autentique en el servidor. Recuerde que el cliente es el dispositivo de acceso (switch, punto de acceso o puerta de enlace VPN), no la PC o la computadora portátil del usuario. Un flujo de trabajo de autenticación RADIUS genérico procede de la siguiente manera:

1. El dispositivo del usuario (el solicitante) establece una conexión con el dispositivo NAS, como un punto de acceso, un switch o un servidor de acceso remoto. 
2. 2.El NAS solicita al usuario sus credenciales de autenticación. RADIUS es compatible con PAP, CHAP y EAP. La mayoría de las implementaciones ahora usan EAP, ya que PAP y CHAP no son seguros. Si se requieren credenciales EAP, el NAS permite que el solicitante transmita datos EAP sobre LAN (EAPoL), pero no cualquier otro tipo de tráfico de red.
3. 3.El solicitante envía las credenciales como datos EAPoL. El cliente RADIUS utiliza esta información para crear un paquete RADIUS de solicitud de acceso, cifrado mediante el secreto compartido. Envía la solicitud de acceso al servidor AAA a través del UDP en el puerto 1812 (de manera predeterminada).
4. 4.El servidor AAA descifra la solicitud de acceso mediante el secreto compartido. Si la solicitud de acceso no se puede descifrar (porque el secreto compartido no se configuró correctamente, por ejemplo), el servidor no responde.
5. 5.Con EAP, habrá un intercambio de paquetes de desafío de acceso y solicitud de acceso a medida que se configure el método de autenticación y se verifiquen las credenciales. El NAS actúa como un sistema de paso, toma mensajes RADIUS del servidor y los encapsula como EAPoL para transmitirlos al solicitante.
6. 6.Al final de este intercambio, si el solicitante se autentica, el servidor AAA responde con un paquete de aceptación de acceso; de lo contrario, se devuelve un paquete de rechazo de acceso.
Optativamente, el NAS puede usar RADIUS para la contabilidad (registro). La contabilidad utiliza el puerto 1813. El servidor de contabilidad puede ser diferente del servidor de autenticación.
