Obfuscation (ofuscación) es el arte de hacer que un mensaje o dato sea difícil de encontrar. Se trata de seguridad por oscuridad, lo que normalmente es obsoleto. Sin embargo, existen algunos usos para las tecnologías de obfuscation (ofuscación): 

Steganography (esteganografía) (que literalmente significa “escritura oculta”) integra información dentro de una fuente inesperada; un mensaje oculto en una imagen, por ejemplo. El documento o archivo contenedor se denomina texto de portada. El mensaje se puede cifrar por medio de algún mecanismo antes de incrustarlo, a fin de proporcionar confidencialidad. La tecnología también puede proporcionar integridad o no repudio; por ejemplo, podría demostrar que algo se imprimió en un dispositivo concreto en un momento determinado, lo que podría demostrar que era auténtico o falso, según el contexto. 
Data masking (enmascaramiento de datos) puede significar que todo o parte del contenido de un campo de base de datos se redacta sustituyendo todas las cadenas de caracteres por “x”, por ejemplo. Un campo puede suprimirse parcialmente para preservar los metadatos con fines de análisis. Por ejemplo, en un número de teléfono, se puede conservar el prefijo de marcación, pero suprimirse el número de abonado. El data masking (enmascaramiento de datos) también puede utilizar técnicas para preservar el formato original del campo. 
Tokenization (tokenización) significa que todo o parte del valor de un campo de base de datos se reemplaza por un token generado aleatoriamente. El token se almacena con el valor original en un servidor de tokens o bóveda de tokens, separado de la base de datos de producción. Si es necesario, una consulta o aplicación autorizadas pueden recuperar el valor original de la bóveda, por lo que la tokenization (tokenización) es reversible. La tokenization (tokenización) se utiliza como sustituto del cifrado porque, desde una perspectiva regulatoria, un campo cifrado tiene el mismo valor que los datos originales.
El data masking (enmascaramiento de datos) y la tokenization (tokenización) se utilizan para la desidentificación. La desidentificación ofusca los datos personales de las bases de datos para que puedan compartirse sin comprometer la privacidad.
